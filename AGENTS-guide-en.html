Contributing

Repository

Repository

All contributions go through GitHub. The main branch is protected — direct pushes are not allowed. You must open a Pull Request.

https://github.com/germanvaras/claude-summary.git

Step-by-step workflow

  1. 1
    Clone the repository 
    git clone https://github.com/germanvaras/claude-summary.git
cd claude-summary
  2. 2
    Create a new branch — required, main is protected

    Use a descriptive name that reflects what you're changing.

    # For a new section
git checkout -b feat/add-rate-limits

# For a fix
git checkout -b fix/hooks-outdated-flags

# For a translation sync
git checkout -b fix/hooks-spanish-sync
  3. 3
    Make your changes using the agents 
    claude
/content-writer add rate-limits   # or update / patch / translate
/guardian --fix rate-limits        # always validate before committing
/exit
  4. 4
    Commit your changes 
    git add en/sections/rate-limits.html es/sections/rate-limits.html \
        en/index.html es/index.html
git commit -m "feat: add rate-limits section"
  5. 5
    Push your branch and open a Pull Request 
    git push origin feat/add-rate-limits

    Then go to github.com/germanvaras/claude-summary and click "Compare & pull request".

Main branch is protected

Pushing directly to main will be rejected. Always work on a separate branch and submit changes via Pull Request. PRs are reviewed before merging.

Add Update Patch Translate

Follow the wizard below to run your first agent command in under 2 minutes:

Install Claude Code

You only need to do this once. Requires Node.js ≥ 18.

npm install -g @anthropic-ai/claude-code

Verify the install

claude --version

You should see a version number like 1.x.x.

Open a Claude Code session in this project

Skills load automatically when the session starts inside the project folder.

cd claude-summary
claude
  1. ✓
    Wait for the session prompt

    You'll see a > prompt inside the Claude Code session.

  2. 2
    Confirm skills are loaded

    Type /skills — you should see guardian and content-writer listed.

Run Guardian — Validate a section

Inside the Claude Code session, type one of the following:

# Validate a single section (e.g. hooks)
/guardian hooks

# Or validate everything
/guardian

Guardian will fetch the official Anthropic docs, compare them to your section files, and return a report with ✅ / ⚠️ / ❌ items.

# Found issues? Apply fixes automatically:
/guardian --fix hooks

Run Content Writer — Add or update content

Still inside the same Claude Code session:

# Add a brand-new section
/content-writer add rate-limits

# Rewrite an outdated section
/content-writer update hooks

# Add one specific piece of info
/content-writer patch hooks --add "PreCompact event"

# Sync the Spanish translation
/content-writer translate hooks

Content Writer will ask for confirmation before writing files. Always follow up with /guardian SLUG to validate what was generated.

Exit the session and commit your changes

# End the Claude Code session
/exit

# Stage the changed files (example: hooks section)
git add en/sections/hooks.html es/sections/hooks.html

# Commit
git commit -m "fix(hooks): update to match current official docs"

Tip: Guardian suggests the exact git add line

After running /guardian --fix, the agent will print the exact git add command for the files it changed. Just copy-paste it.

Guardian Agent

Validation

What it does

Validates documentation sections against official Anthropic sources. Detects outdated CLI flags, wrong model names, incorrect behavior descriptions, and missing content.

Commands

/guardian

Full audit — validates all sections

/guardian SLUG

Validate a single section by slug

/guardian SLUG1 SLUG2

Validate multiple sections at once

/guardian --fix SLUG

Validate AND apply corrections (asks for confirmation)

# Full audit
/guardian

# Single section
/guardian hooks

# Multiple sections
/guardian models tokens mcp

# Validate and fix
/guardian --fix hooks

How it works — 4 steps

  1. 1
    Fetches the relevant page

    Downloads the section from docs.anthropic.com

  2. 2
    Reads your section files

    Reads both en/sections/SLUG.html and es/sections/SLUG.html

  3. 3
    Compares factual claims

    CLI flags, model names, context windows, event names, config keys

  4. 4
    Produces a report

    Three categories: ✅ Correct · ⚠️ Outdated · ❌ Missing

Report categories

✅
Correct

Matches official documentation exactly

⚠️
Outdated / Inaccurate

Exists but doesn't match current official version

❌
Missing

In official docs but absent from your section

Example report output

## Guardian Report: hooks
Validated: 2026-03-28
Official source: https://docs.anthropic.com/en/docs/claude-code/hooks

### ✅ Correct
- PostToolUse event name — matches official docs
- Hook timeout default (60s) — correct

### ⚠️ Outdated / Inaccurate
- "stdin" payload field listed as "input" → official docs use "stdin"
  File: en/sections/hooks.html (look for: "input")

### ❌ Missing
- PreCompact hook event not documented (added in recent release)

| Section | Status |  Issues                 |
|---------|--------|-------------------------|
| hooks   | ⚠️     | 1 outdated, 1 missing   |

Content Writer Agent

Create / Update

What it does

Creates new bilingual sections or updates existing ones, always sourced from official Anthropic documentation. Handles both English and Spanish files automatically.

Commands

/content-writer add SLUG

Create a brand-new bilingual section from scratch

/content-writer update SLUG

Rewrite an existing section from current official docs

/content-writer patch SLUG --add "X"

Insert one specific piece of info into a section

/content-writer translate SLUG

Sync the Spanish version with the English version (or vice versa)

# Add a completely new section
/content-writer add rate-limits

# Rewrite an existing section
/content-writer update hooks

# Add one specific piece of info
/content-writer patch hooks --add "PreCompact event"

# Sync Spanish ↔ English
/content-writer translate hooks

What each subcommand does

CommandActionOutput
add SLUG New section Creates en/sections/SLUG.html + es/sections/SLUG.html, updates nav in both index.html
update SLUG Full rewrite Shows diff vs official docs, asks confirmation, rewrites EN + ES
patch SLUG --add "X" Targeted addition Inserts new content in the right place in both language files
translate SLUG Sync languages Identifies what's in one language but not the other, translates the gap

Combined Workflows

End-to-end

1. Add a new section (after an Anthropic release)

# Start session
cd claude-summary && claude

# Create the new section
/content-writer add memory-management

# Validate immediately
/guardian memory-management

# Apply any fixes Guardian found
/guardian --fix memory-management

# Exit and commit
/exit
git add en/sections/memory-management.html es/sections/memory-management.html \
        en/index.html es/index.html
git commit -m "feat: add memory-management section"

2. Fix outdated content in an existing section

claude
/content-writer update hooks
/guardian --fix hooks
/exit
git add en/sections/hooks.html es/sections/hooks.html
git commit -m "fix(hooks): update to match current official docs"

3. Add one specific piece of info (patch)

claude
/content-writer patch hooks --add "PreCompact event"
/guardian --fix hooks
/exit
git add en/sections/hooks.html es/sections/hooks.html
git commit -m "fix(hooks): add PreCompact event documentation"

4. Sync translations

claude
/content-writer translate hooks
/guardian hooks   # optional: verify synced version
/exit
git add es/sections/hooks.html
git commit -m "fix(hooks): sync Spanish translation with English"

5. Guardian-only fix (no Content Writer needed)

claude
/guardian --fix models
/exit
# Guardian prints the exact files to stage
git commit -m "fix(models): correct outdated model names and context windows"

Troubleshooting

Common issues
Skills don't appear in /skills
Close the session with /exit and reopen it — skills load at startup only.
WebFetch fails for docs.anthropic.com
The site may be temporarily unavailable. Try again in a few minutes. Guardian will automatically fall back to WebSearch if the fetch fails.
Content Writer creates malformed HTML
Run Guardian immediately after: /guardian SLUG. It will identify structural issues. You can also validate manually with browser dev tools (F12 → Console).
Agent edits the wrong file
Check that the slug you passed matches the filename exactly. Slugs are case-sensitive on Linux/Mac file systems.

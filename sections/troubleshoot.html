🔧 Troubleshooting Errores comunes

📌
Los problemas más frecuentes al trabajar con Claude Code, organizados por síntoma. Hacé clic en cada uno para ver la causa y solución.

🧩 Skills

Mi skill no se activa cuando la pido Skill ▶
🔴
Síntoma: Claude no activa la skill aunque el pedido parece relevante.

Causas y soluciones:

  • 🔄 No reiniciaste Claude Code después de crear o editar la skill. Las skills cargan al inicio de la sesión — siempre cerrá con /exit y reabrí con claude tras cambios.
  • 📝 La descripción es muy vaga. Si dice solo "ayuda con código", Claude no sabe cuándo usarla. Agregá verbos y frases concretas: "Use when reviewing PRs, checking code before merge, or when asked for feedback on a branch."
  • 📁 El archivo está mal ubicado. La estructura debe ser .claude/skills/nombre-skill/SKILL.md — no .claude/skills/SKILL.md directamente. Cada skill necesita su propia subcarpeta.
  • ✏️ El frontmatter tiene errores. Los campos name y description son obligatorios y deben estar entre los dashes --- exactamente.

Verificación rápida:

/skills

Si tu skill no aparece en la lista, el problema es de ubicación o frontmatter. Si aparece pero no se activa, el problema es la descripción.

Se activa la skill incorrecta cuando tengo dos con nombres similares Skill ▶
🔴
Síntoma: Claude activa la skill del proyecto en vez de la personal, o viceversa.

El orden de prioridad es: Managed → Personal → Project → Plugins. Si tenés dos skills con el mismo name, gana la de mayor prioridad.

  • Usá nombres descriptivos para evitar colisiones: frontend-review en vez de review.
  • Si querés que la de proyecto sobreescriba la personal, la única opción es renombrar la personal o darle un name distinto.
  • Usá /skills para ver el origen (personal/proyecto) de cada skill activa.
La skill carga pero Claude no sigue las instrucciones correctamente Skill ▶
🔴
Síntoma: Claude confirma que activó la skill pero el resultado no respeta el formato o los pasos definidos.
  • 📏 SKILL.md es demasiado largo. Si supera las 500 líneas, Claude puede omitir partes. Mové el material extenso a archivos en references/ y referencialos desde el SKILL.md.
  • ⚔️ Conflicto con CLAUDE.md. Si CLAUDE.md tiene instrucciones que contradicen la skill, pueden competir. Revisá que no haya solapamiento.
  • 🔢 Instrucciones ambiguas. Los pasos numerados y las listas con viñetas funcionan mejor que los párrafos largos para instrucciones procedurales.

🪝 Hooks

El hook no se ejecuta después de que Claude edita un archivo Hook ▶
🔴
Síntoma: Claude escribe o edita archivos pero el hook no dispara.
  • 🔄 Reiniciá Claude Code. Los hooks se leen al iniciar la sesión.
  • 📄 Verificá el JSON del settings.json. Un error de sintaxis silencia todos los hooks. Validá con:
claude config list
  • 🎯 Revisá el matcher. "matcher": "Write" solo activa en writes. Para edits también, usá "matcher": "Write|Edit".
  • 📁 Ubicación del settings.json. Debe estar en .claude/settings.json (proyecto) o ~/.claude/settings.json (global).
El hook falla con "command not found" o errores de path Hook ▶
🔴
Síntoma: El hook dispara pero el comando no se encuentra o falla con error de ruta.
  • 🛣️ Los hooks corren con el PATH del sistema, no del shell interactivo. Usá rutas absolutas. En Windows, verificá con where npm la ruta completa.
  • 📦 El comando requiere estar en el directorio del proyecto. Prefijá: cd C:\ruta\proyecto && npm run lint
  • 🔑 Permisos de ejecución. En Linux/macOS, verificá: chmod +x script.sh
  • 🔍 Usá modo verbose para ver el error exacto:
claude --verbose

🔌 MCP

El MCP server aparece como desconectado MCP ▶
🔴
Síntoma: /mcp muestra el server como offline o Claude dice que no puede conectarse.
  • 📦 El paquete no está instalado. Con npx -y se instala automáticamente, pero puede fallar en redes corporativas. Instalalo manualmente primero: npm install -g @modelcontextprotocol/server-NOMBRE
  • 🔑 Variables de entorno faltantes. Verificá que las variables en el bloque env del settings.json estén definidas y sean correctas.
  • 🔒 Problema de conectividad. Si la DB o API es remota, verificá que el host sea accesible desde tu máquina.
  • 📋 Revisá los logs del server con verbose:
claude --verbose

En modo verbose verás el error de inicialización del MCP server con detalle.

📄 CLAUDE.md / General

Claude ignora las instrucciones de CLAUDE.md CLAUDE.md ▶
🔴
Síntoma: Claude no respeta convenciones o restricciones definidas en CLAUDE.md.
  • 📏 CLAUDE.md es demasiado largo. Si tiene cientos de líneas, las instrucciones del final pueden quedar fuera del foco efectivo. Mantenelo bajo 200 líneas y mové el material extenso a skills.
  • ⚔️ Instrucción contradictoria en el chat. Si en el mismo mensaje das una instrucción contraria a CLAUDE.md, puede prevalecer. CLAUDE.md establece defaults, no reglas absolutas. Para restricciones absolutas, usá deny en permisos.
  • 📁 Archivo mal nombrado o mal ubicado. Debe ser exactamente CLAUDE.md (mayúsculas) en la raíz del proyecto, o en .claude/CLAUDE.md.
  • 💡 El contexto se llenó. En sesiones largas, CLAUDE.md puede perder efectividad. Usá /compact para comprimir el historial.
El contexto se llena rápido y Claude empieza a "olvidar" cosas General ▶
🔴
Síntoma: En sesiones largas, Claude empieza a ignorar instrucciones o perder el hilo de la tarea.
  • 🗜️ Usá /compact para comprimir el historial de la conversación sin perder el contexto esencial. Podés agregar instrucciones de foco: /compact focalizá en los cambios del módulo Orders
  • 📂 Mové contenido de CLAUDE.md a skills. Lo que no necesita estar siempre presente consume contexto innecesariamente.
  • ✂️ Dividí tareas largas. En vez de una sesión gigante, usá subagentes o sesiones separadas para módulos distintos.
  • 🔄 Empezá una sesión nueva con solo claude para tener un contexto limpio. CLAUDE.md y las skills se recargan automáticamente. Usá claude -c para retomar la sesión anterior (no comprime — continúa donde estaba).

🔑 Autenticación

Claude Code pide login cada vez que lo abro Auth ▶
🔴
Síntoma: El token de sesión no persiste entre sesiones.
  • 📁 El archivo de credenciales puede haber sido borrado. Verificá que existe: C:\Users\<usuario>\.claude\credentials.json
  • 🔒 Permisos del archivo. El proceso de Claude Code debe poder leer y escribir ese archivo.
  • 🔑 Alternativa permanente: Configurá ANTHROPIC_API_KEY como variable de entorno del sistema con setx. No necesita login por navegador.

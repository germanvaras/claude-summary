Idea clave: Elegir el modelo correcto impacta velocidad, calidad y costo. La regla general: Opus para lo complejo y crítico, Sonnet para el desarrollo diario, Haiku para tareas simples y rápidas.

⚙️ Cómo cambiar el modelo

El modelo por defecto en Claude Code es Sonnet. Podés cambiarlo de varias formas:

MétodoComandoAlcance
Durante la sesión/model claude-opus-4-6Solo esa sesión
Al iniciarclaude --model claude-opus-4-6Solo esa sesión
En settings.json"model": "claude-sonnet-4-6"Permanente (User o Project scope)
En frontmatter de skillmodel: claude-haiku-4-5-20251001Solo cuando esa skill está activa
📌
IDs exactos de modelo: claude-opus-4-6 · claude-sonnet-4-6 · claude-haiku-4-5-20251001. Usá el ID completo en configuración para evitar ambigüedad.
⚠️
Haiku es versión 4.5, no 4.6 — no son todos de la misma generación. El sufijo de fecha en claude-haiku-4-5-20251001 es parte del ID oficial y es obligatorio. Usar claude-haiku-4-5 sin la fecha o intentar claude-haiku-4-6 puede fallar silenciosamente o hacer fallback al modelo por defecto.

💰 Tokens y costo en la práctica

En Claude Code, el costo se mide en tokens consumidos — tanto de entrada (historial, archivos, skills, CLAUDE.md) como de salida (lo que Claude genera). Cuanto más compleja la tarea y más contexto acumulado, más tokens consume cada turno.

📌
Input y output no tienen el mismo precio. Los tokens de salida cuestan significativamente más por token que los de entrada. En pipelines donde Claude genera respuestas largas (reportes, código extenso, análisis), el costo real está dominado por el output — no por el contexto de entrada. Optimizá también el tamaño de las respuestas, no solo el contexto.
EscenarioModelo idealPor qué
Sesión interactiva de desarrolloSonnetMuchos turnos → costo acumulado importa
Una tarea crítica puntual (arquitectura, seguridad)OpusPocos turnos → el costo extra se justifica
Agente autónomo en pipeline CI/CDSonnetMuchos turnos autónomos con intervención mínima
Tareas repetitivas: formateo, renames, boilerplateHaikuNo hace falta potencia máxima → costo mínimo
⚠️
No uses Opus para todo. Una sesión de refactoring masivo con Opus puede costar 10–20× más que con Sonnet para un resultado similar en tareas estándar. Reservá Opus para cuando realmente importa la máxima inteligencia.

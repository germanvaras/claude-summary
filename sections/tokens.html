🪙 Tokens y Contexto Fundamental

💡
Idea clave: Entender tokens y contexto es entender los límites reales de Claude. Todo lo demás — skills, CLAUDE.md, subagentes — existe en parte para gestionar este recurso finito.

🔢 ¿Qué es un token?

Un token es la unidad mínima que el modelo procesa. No es una palabra ni un carácter — es un fragmento que puede ser parte de una palabra, una palabra entera o un signo de puntuación. El tokenizer divide el texto antes de enviarlo al modelo.

📌
El tokenizer de Claude está optimizado para inglés. El mismo texto en español o en otros idiomas generalmente consume más tokens que en inglés — es decir, es menos eficiente.

📦 El context window

El context window es el "espacio de trabajo" del modelo — todo lo que Claude puede "ver" a la vez: el historial de la conversación, archivos leídos, CLAUDE.md, skills activas y resultados de herramientas. Es finito. Todo comparte el mismo espacio.

📌
Los modelos actuales de Claude tienen distintos context windows: Opus 4.6 y Sonnet 4.6 tienen 1.000.000 tokens (1M); Haiku 4.5 tiene 200.000 tokens. Este presupuesto cubre el historial, archivos, CLAUDE.md y skills (input) más la respuesta generada (output). En uso interactivo raramente es un problema. En pipelines con generación masiva, el límite de output por respuesta es el techo práctico real: Opus 4.6 hasta 128K tokens, Sonnet 4.6 y Haiku 4.5 hasta 64K tokens.

🫧 ¿Qué pasa cuando el contexto se llena?

Claude no te avisa con un error explícito — lo que pasa es más sutil y difícil de detectar:

1

Degradación gradual

Claude empieza a perder el foco en instrucciones que estaban al principio de la conversación. CLAUDE.md puede quedar fuera del "foco efectivo" aunque técnicamente siga cargado. Las respuestas se vuelven más genéricas y menos precisas.

2

Pérdida de coherencia

Claude empieza a contradecirse: aplica convenciones distintas a las establecidas, repite preguntas que ya respondiste, o ignora restricciones de CLAUDE.md que definiste al inicio de la sesión.

3

Corte forzado

En casos extremos, la API rechaza la request porque supera el límite máximo. Aparece un error explícito. Es el caso menos frecuente.

⚠️
El "olvido" de Claude en sesiones largas casi nunca es un bug del modelo — es el contexto cargado. Antes de asumir que Claude "no entiende", usá /compact para comprimir el historial, o iniciá una sesión nueva con solo claude. Usá claude -c solo si querés retomar la sesión anterior donde quedó — no es una sesión nueva.

🗜️ /compact — la herramienta clave

El comando /compact comprime el historial de la conversación: el modelo genera un resumen denso de lo que se habló y reemplaza el historial completo por ese resumen. El resultado: libera espacio de contexto sin perder el hilo de la tarea.

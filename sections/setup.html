⚙️ Instalación CLI Windows

💡
Idea clave: Claude Code es una CLI que corre en la terminal. Una vez instalada globalmente, la ejecutás desde cualquier directorio — VS Code, Visual Studio, Windows Terminal o PowerShell.

📋 Requisitos previos

RequisitoNotasVerificación
Cuenta AnthropicSuscripción Claude Pro o Anthropic Console (API key)claude.ai / console.anthropic.com
Git for WindowsRecomendado para proyectos con control de versionesgit --version
Node.js 18+Solo necesario si instalás vía npm. El instalador nativo lo incluye.node --version

🚀 Instalación

1

Método A — Instalador nativo (recomendado para Windows)

Abrí PowerShell y ejecutá:

irm https://claude.ai/install.ps1 | iex

El instalador nativo incluye sus propias dependencias — no necesitás instalar Node.js por separado. Se actualiza automáticamente en segundo plano.

Alternativa con WinGet:

winget install Anthropic.ClaudeCode
⚠️
Si PowerShell bloquea la ejecución de scripts, ejecutá primero: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser y luego volvé a correr el instalador.
2

Método B — npm (si ya tenés Node.js 18+)

npm install -g @anthropic-ai/claude-code

Con este método, claude se instala en %APPDATA%\npm. Si el comando no se reconoce después de instalar, verificá que esa ruta esté en el PATH del sistema.

3

Verificar la instalación

claude --version

O desde dentro de una sesión, usá el slash command:

/doctor

Verifica versión, permisos y conectividad con la API. Si algo está mal, lo reporta con detalle.

4

Primera ejecución y autenticación

Navegá a cualquier proyecto y ejecutá:

claude

La primera vez te pedirá autenticación. Ver sección 🔑 Autenticación para las opciones disponibles (navegador o API key).

🖥️ Ejecutar desde cualquier lugar en Windows

Claude Code usa el directorio donde lo ejecutás como raíz del proyecto. Lo primero siempre es posicionarte en la carpeta correcta antes de correr claude.

TerminalCómo abrir en el directorio del proyecto
VS CodeTerminal integrado (Ctrl+`) — abre directamente en el workspace. Solo escribís claude.
Visual StudioMenú View → Terminal (VS 2022+) — abre en el directorio de la solución. Escribís claude ahí.
Windows Terminal / PowerShellcd C:\ruta\al\proyecto y después claude.
Explorador de WindowsClic en la barra de direcciones de la carpeta del proyecto → escribís powershell → Enter. Abre PowerShell directo en ese directorio.
⚠️
Si claude no se reconoce como comando después de instalar: cerrá y volvé a abrir la terminal. Windows no recarga el PATH automáticamente en sesiones ya abiertas.

🔧 Problemas comunes en Windows

ProblemaCausaSolución
claude no encontrado (instalación npm) PATH no incluye la carpeta npm global Cerrá y abrí la terminal. Si persiste: verificá que %APPDATA%\npm esté en el PATH del sistema (Panel de control → Variables de entorno).
claude no encontrado (instalador nativo) PATH no recargado en la sesión actual Cerrá y abrí la terminal. Si persiste, reinstalá con el script oficial.
El script de instalación falla en PowerShell Política de ejecución de scripts bloqueada Ejecutá primero: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser. Luego volvé a correr el instalador.
Falla en redes corporativas Proxy bloqueando el script o la API de Anthropic Usá WinGet: winget install Anthropic.ClaudeCode. Para la API, consultá con el área de infraestructura sobre el proxy y la autenticación via API key.
Visual Studio no encuentra claude El terminal de VS puede tener un PATH diferente al del sistema Usá el terminal integrado de VS Code, o abrí PowerShell externo, navegá al directorio del proyecto con cd y ejecutá claude desde ahí.

📁 Verificar que Claude detectó tu proyecto

Una vez dentro de la sesión, estos comandos confirman que Claude cargó el contexto correcto:

/statusMuestra el estado de configuración: directorio activo, modelo, skills cargadas
/skillsLista las skills disponibles para el proyecto actual
/mcpEstado de los MCP servers configurados
/doctorDiagnóstico completo del entorno (versión, permisos, conectividad)
✅
Si /skills lista las skills del proyecto y /doctor no reporta errores, Claude está listo para trabajar.

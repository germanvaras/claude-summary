🔑 Autenticación Configuración

💡
Idea clave: Claude Code soporta dos métodos de autenticación: login por navegador (para uso interactivo) y API key via variable de entorno (para CI/CD, servidores o entornos sin GUI).

🌐 Método 1 — Login por navegador (uso interactivo)

Es el método por defecto. Al ejecutar claude por primera vez, abre el navegador para que inicies sesión con tu cuenta de Anthropic.

1

Ejecutar Claude Code

claude

Si no hay sesión activa, muestra un mensaje y abre el navegador automáticamente.

2

Aprobar en el navegador

Iniciás sesión en claude.ai con tu cuenta Anthropic y aprobás el acceso para Claude Code.

3

Token guardado localmente

El token de sesión se guarda en:

OSUbicación
WindowsC:\Users\<usuario>\.claude\credentials.json
macOS / Linux~/.claude/credentials.json

Las ejecuciones posteriores reutilizan este token automáticamente — no necesitás repetir el login.

⚠️
Si el token expira o la cuenta cambia, ejecutá claude logout y volvé a hacer login. En entornos compartidos (servidores, CI), nunca uses el login por navegador — usá API key.

🔐 Método 2 — API Key (CI/CD, servidores, entornos sin GUI)

Si establecés la variable de entorno ANTHROPIC_API_KEY, Claude Code la usa automáticamente sin necesitar login por navegador. Este método es obligatorio en entornos automatizados.

Obtener una API key

En console.anthropic.com → API Keys → Create Key. Las keys comienzan con sk-ant-api03-...

Configurar en Windows

Sesión actual solamente:

$env:ANTHROPIC_API_KEY = "sk-ant-api03-..."

Permanente para tu usuario:

setx ANTHROPIC_API_KEY "sk-ant-api03-..."

Después de setx, cerrá y reabrí la terminal para que tome efecto.

Verificar que Claude usa la key

claude --version   # debe responder sin pedir login
/doctor            # muestra "API key: configurada"
📌
Si tanto ANTHROPIC_API_KEY como credentials.json están presentes, la variable de entorno tiene prioridad.

🔒 Buenas prácticas de seguridad

ContextoMétodo recomendadoPor qué
Desarrollo local interactivoLogin por navegadorSin credenciales en archivos ni variables
CI/CD (GitHub Actions, etc.)Secret del repositorio → env varLa key no se expone en logs ni en el código
Servidor de equipoVariable de entorno del sistema, gestionada por ITControl centralizado, no en código
Docker / containersVariable de entorno inyectada al rundocker run -e ANTHROPIC_API_KEY=... ...
🔴
Nunca hagas esto:
  • Guardar la API key en settings.json del proyecto (va al repositorio)
  • Hardcodear la key en scripts o código fuente
  • Compartir ~/.claude/credentials.json

🚀 Primer uso — flujo completo desde cero

Una vez autenticado, este es el flujo típico para empezar a trabajar en un proyecto:

# 1. Ir al directorio del proyecto
cd C:\mis-proyectos\mi-api

# 2. Iniciar Claude Code
claude

# 3. Si el proyecto no tiene CLAUDE.md, generarlo
/init

# 4. Verificar que todo cargó correctamente
/status
/skills

# 5. Hacer la primera tarea
"explicá la arquitectura del proyecto"
✅
Con esto ya estás operativo. El CLAUDE.md generado por /init le da a Claude el contexto del proyecto desde el primer mensaje.

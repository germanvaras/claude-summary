Antes de meterte en los detalles: entendé qué hace cada pieza. Si esto queda claro, el resto de la documentación va a encajar solo.

🗺️ El mapa completo

PiezaRolCuándo aplica
📄 CLAUDE.mdMemoria persistente del proyectoSiempre — carga en cada conversación automáticamente
🧩 SkillsConocimiento especializado activableCuando la tarea coincide semánticamente con la descripción de la skill
🪝 HooksAutomatizaciones disparadas por eventosCuando Claude hace algo: escribe un archivo, ejecuta un comando, termina
🔌 MCP ServersConexión a herramientas externasCuando necesitás APIs, bases de datos o servicios de terceros
🔐 PermisosGuardarraíles de seguridadDefine qué puede y qué no puede hacer Claude en cada contexto
⚡ Slash CommandsAcciones manuales explícitasCuando vos decidís ejecutar algo, con control total del momento

🔗 Cómo se relacionan

Cada conversación con Claude Code
│
├── Carga automática al iniciar
│   ├── CLAUDE.md          → contexto permanente del proyecto
│   └── Skills relevantes  → conocimiento especializado por tarea
│
├── Se disparan solos (event-driven)
│   └── Hooks              → reaccionan a lo que Claude hace
│
├── Vos los activás manualmente
│   └── Slash Commands     → /deploy, /changelog, /standup
│
├── Disponibles como herramientas
│   └── MCP Servers        → APIs, DBs, servicios externos
│
└── Guardarraíles siempre activos
    └── Permisos           → qué puede y qué no puede hacer Claude

🎯 Regla de decisión rápida

NecesidadSolución
Claude no sabe nada del proyecto📄 Escribí un CLAUDE.md con el contexto
Claude no sabe cómo hacer algo🧩 Creá una Skill con las instrucciones
Querés que algo pase automáticamente🪝 Configurá un Hook para ese evento
Necesitás conectar con una API o DB🔌 Agregá un MCP Server
Querés ejecutar algo cuando vos querés⚡ Creá un Slash Command
Claude hace cosas que no debería🔐 Configurá Permisos en settings.json
✅
Estas piezas son complementarias, no alternativas. Un proyecto bien configurado usa todas: CLAUDE.md da el contexto, las skills el conocimiento, los hooks la automatización, MCP las conexiones externas y los permisos la seguridad.

🌿 Worktrees Aislamiento

💡
Idea clave: Un worktree es una copia de trabajo aislada del mismo repositorio Git. Claude Code puede trabajar en un worktree sin tocar tu directorio principal — ideal para refactoring agresivo, experimentos o trabajo agéntico de alto impacto.

🌳 ¿Qué es un git worktree?

Git permite tener múltiples directorios de trabajo vinculados al mismo repositorio. Cada worktree tiene su propia rama y estado de archivos, pero comparten el historial. Es como tener el proyecto "clonado" en otra carpeta, pero mucho más eficiente.

# Sin worktrees — Claude trabaja directamente en tu directorio
cd C:\proyectos\mi-api
claude
> "refactorizá el módulo Orders completo"
# Claude modifica archivos en tu directorio de trabajo

# Con worktrees — Claude trabaja en una copia aislada
cd C:\proyectos\mi-api
claude --worktree
> "refactorizá el módulo Orders completo"
# Claude trabaja en una copia separada. Tu directorio no cambia.

🚀 Cómo usar worktrees con Claude Code

1

Iniciar Claude en un worktree aislado

claude --worktree

Claude Code crea automáticamente un nuevo worktree desde la rama actual, inicia la sesión en ese worktree y trabaja allí.

2

Claude trabaja en el worktree

Desde tu directorio principal podés seguir trabajando normalmente — los cambios de Claude están en la copia aislada. Al terminar la sesión, Claude Code te informa qué archivos cambió en el worktree.

3

Revisar y decidir qué hacer con los cambios

# Ver los cambios en el worktree
git diff HEAD..worktree/branch

# Si te gustaron los cambios — mergear o crear PR
git merge worktree/branch

# Si no te gustaron — descartar el worktree
git worktree remove /ruta/al/worktree

🎯 Cuándo usar worktrees

EscenarioPor qué worktree
Refactoring grande y riesgosoSi Claude se equivoca, tu rama principal no se contamina. Podés descartar el worktree y empezar de nuevo.
Experimentar con arquitecturaProbás un enfoque sin comprometer tu trabajo en progreso.
Agente autónomo de larga duraciónEl agente trabaja en paralelo en el worktree mientras vos seguís en la rama principal.
CI/CD con review humanoClaude genera los cambios en una rama separada via worktree, vos revisás y aprobás antes de mergear.

⚖️ Con worktree vs. sin worktree

Sin worktreeCon --worktree
¿Dónde trabaja Claude?Directamente en tu directorioEn una copia aislada
¿Afecta tu trabajo actual?Sí — modifica los archivos que tenés abiertosNo — tu directorio permanece intacto
¿Podés descartar todo?Solo con git reset o stashSí — simplemente eliminás el worktree
¿Podés revisar antes de aceptar?Necesitás revertir manualmenteSí — ves el diff del worktree y decidís
OverheadNingunoOcupa espacio en disco extra (una copia de los archivos)
📌
Los worktrees requieren que el proyecto esté bajo control de versiones Git. Si el proyecto no tiene repositorio Git, el flag --worktree no puede crear una copia aislada.

🛠️ Comandos Git útiles con worktrees

# Ver todos los worktrees activos
git worktree list

# Eliminar un worktree (si ya no lo necesitás)
git worktree remove /ruta/al/worktree

# Limpiar worktrees huérfanos
git worktree prune

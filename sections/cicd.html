🔄 Claude Code en CI/CD Automatización

💡
Idea clave: Claude Code tiene un modo headless (no interactivo) diseñado para pipelines de CI/CD y automatizaciones. Se ejecuta, completa la tarea y sale — sin sesión interactiva, con output estructurado.

🚀 Modo headless — claude -p

El flag -p (print / headless) envía un prompt, espera la respuesta completa y sale. No abre una sesión interactiva.

# Sintaxis básica
claude -p "tarea que querés ejecutar"

# Ejemplo: generar changelog automáticamente
claude -p "generá el changelog para la versión 2.4.0 basándote en los commits desde el tag v2.3.0"

# Ejemplo: revisar código antes de merge
claude -p "revisá los archivos modificados en este PR y listá cualquier problema crítico"

📤 Formatos de output

En modo headless podés controlar el formato de salida:

FlagFormatoCuándo usarlo
--output-format textTexto plano (por defecto)Scripts que leen la salida como string
--output-format jsonJSON con la respuesta completa al finalIntegración con herramientas que parsean JSON
--output-format stream-jsonEventos JSON en streaming, uno por línea (NDJSON)Procesar la respuesta en tiempo real mientras llega
 
# Output JSON para parsing en scripts
claude -p "listá las clases con cobertura bajo el 80%" --output-format json

# Stream para procesar en tiempo real
claude -p "refactorizá OrderService.cs" --output-format stream-json | jq '.content'

🔒 Autenticación en CI/CD

En pipelines sin interfaz gráfica, el login por navegador no funciona. Siempre usá API key:

# GitHub Actions — secret configurado en el repositorio
env:
  ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

# Docker
docker run -e ANTHROPIC_API_KEY=sk-ant-... mi-imagen claude -p "tarea"

# Variables de entorno en el sistema CI
export ANTHROPIC_API_KEY="sk-ant-api03-..."
claude -p "tarea"

⚙️ Flags importantes para CI/CD

--max-turns <n>Limita la cantidad de turnos agénticos. Evita que Claude entre en bucles infinitos. Recomendado: 10–30 dependiendo de la complejidad.
--dangerously-skip-permissionsOmite todas las confirmaciones interactivas. Necesario en CI (no hay usuario para aprobar). Usá con reglas deny en settings.json como guardarraíl.
--allowedTools Read,Write,BashRestringe qué herramientas puede usar Claude en esta ejecución. Principio de mínimo privilegio.
--disallowedTools BashBloquea herramientas específicas para esta ejecución.
--output-format jsonSalida en JSON — más fácil de parsear en scripts de CI.
--verboseLogging detallado turno a turno. Útil para debugging de pipelines.

📋 Ejemplo — GitHub Actions

# .github/workflows/claude-review.yml
name: Claude Code Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  code-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Claude review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude -p "Revisá los archivos modificados en este PR. \
            Listá problemas críticos, sugerencias de mejora y si hay tests faltantes. \
            Formato: lista markdown con severidad [critical/suggestion/nitpick]." \
            --max-turns 20 \
            --allowedTools Read,Grep,Glob \
            --output-format text

📋 Ejemplo — Script de automatización

#!/bin/bash
# generate-release-notes.sh

VERSION=$1
PREV_VERSION=$2

echo "Generando release notes para $VERSION..."

NOTES=$(claude -p "Generá release notes para la versión $VERSION. \
  Basate en los commits desde el tag $PREV_VERSION hasta HEAD. \
  Agrupá por: Features, Bug Fixes, Breaking Changes. \
  Formato Markdown." \
  --max-turns 5 \
  --output-format text)

echo "$NOTES" > "RELEASE_NOTES_$VERSION.md"
echo "Listo: RELEASE_NOTES_$VERSION.md"

🛡️ Buenas prácticas de seguridad en CI/CD

PrácticaPor qué
Usá --allowedTools con mínimo privilegioEn CI, Claude no necesita todas las herramientas. Restringí a lo que realmente necesita la tarea.
Configurá deny rules en settings.json del proyectoLas reglas deny funcionan incluso con --dangerously-skip-permissions
Nunca hardcodees la API key en el código o en archivos commiteadosUsá secrets del CI (GitHub Secrets, etc.)
Usá --max-turns siempre en CIEvita ejecuciones que no terminan y consumen créditos sin control
Revisá el exit code del procesoClaude -p retorna 0 en éxito, distinto de 0 en error — usalo para controlar el pipeline
✅
Combinación ideal para CI/CD: ANTHROPIC_API_KEY en secret + --dangerously-skip-permissions + --allowedTools con mínimo privilegio + --max-turns como límite de seguridad + reglas deny en settings.json del proyecto.

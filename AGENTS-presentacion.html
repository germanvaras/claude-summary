Cómo Contribuir

Repositorio

Repositorio

Todas las contribuciones pasan por GitHub. La rama main está protegida — no se permiten pushes directos. Es obligatorio abrir un Pull Request.

https://github.com/germanvaras/claude-summary.git

Flujo paso a paso

  1. 1
    Clonar el repositorio 
    git clone https://github.com/germanvaras/claude-summary.git
cd claude-summary
  2. 2
    Crear una nueva rama — obligatorio, main está protegida

    Usa un nombre descriptivo que refleje qué estás cambiando.

    # Para una nueva sección
git checkout -b feat/add-rate-limits

# Para una corrección
git checkout -b fix/hooks-flags-desactualizados

# Para sincronizar traducción
git checkout -b fix/hooks-sync-espanol
  3. 3
    Realizar los cambios usando los agentes 
    claude
/content-writer add rate-limits   # o update / patch / translate
/guardian --fix rate-limits        # siempre validar antes de commitear
/exit
  4. 4
    Commitear los cambios 
    git add en/sections/rate-limits.html es/sections/rate-limits.html \
        en/index.html es/index.html
git commit -m "feat: add rate-limits section"
  5. 5
    Pushear la rama y abrir un Pull Request 
    git push origin feat/add-rate-limits

    Luego ir a github.com/germanvaras/claude-summary y hacer clic en "Compare & pull request".

La rama main está protegida

Hacer push directo a main será rechazado. Siempre trabaja en una rama separada y envía los cambios mediante Pull Request. Los PRs son revisados antes de mergearse.

Convenciones para nombres de ramas

Corrección / actualizaciónfix/fix/hooks-flags-desactualizados
Sincronizar traducciónfix/fix/hooks-sync-espanol

Sigue el asistente paso a paso para ejecutar tu primer comando en menos de 2 minutos:

Instalar Claude Code

Solo necesitas hacerlo una vez. Requiere Node.js ≥ 18.

npm install -g @anthropic-ai/claude-code

Verificar la instalación

claude --version

Deberías ver un número de versión como 1.x.x.

Abrir una sesión de Claude Code en este proyecto

Las skills se cargan automáticamente al iniciar la sesión dentro de la carpeta del proyecto.

cd claude-summary
claude
  1. ✓
    Esperar el prompt de la sesión

    Verás un prompt > dentro de la sesión de Claude Code.

  2. 2
    Confirmar que las skills están cargadas

    Escribe /skills — deberías ver guardian y content-writer en la lista.

Ejecutar Guardian — Validar una sección

Dentro de la sesión de Claude Code, escribe uno de los siguientes:

# Validar una sola sección (ej: hooks)
/guardian hooks

# O validar todo el sitio
/guardian

Guardian descargará los docs oficiales de Anthropic, los comparará con tus archivos de sección y devolverá un reporte con ítems ✅ / ⚠️ / ❌.

# ¿Encontró problemas? Aplicar correcciones automáticamente:
/guardian --fix hooks

Ejecutar Content Writer — Agregar o actualizar contenido

Dentro de la misma sesión de Claude Code:

# Agregar una sección completamente nueva
/content-writer add rate-limits

# Reescribir una sección desactualizada
/content-writer update hooks

# Agregar un dato específico
/content-writer patch hooks --add "PreCompact event"

# Sincronizar la traducción al español
/content-writer translate hooks

Content Writer pedirá confirmación antes de escribir archivos. Siempre ejecuta /guardian SLUG después para validar lo generado.

Salir de la sesión y commitear los cambios

# Terminar la sesión de Claude Code
/exit

# Agregar los archivos modificados (ejemplo: sección hooks)
git add en/sections/hooks.html es/sections/hooks.html

# Commitear
git commit -m "fix(hooks): actualizar según docs oficiales"

Tip: Guardian sugiere el git add exacto

Después de ejecutar /guardian --fix, el agente imprime el comando git add exacto para los archivos que modificó. Solo cópialo y pégalo.

Agente Guardian

Validación

¿Qué hace?

Valida secciones de documentación contra las fuentes oficiales de Anthropic. Detecta flags CLI desactualizados, nombres de modelos incorrectos, comportamientos mal descritos y contenido faltante.

Comandos

/guardian

Auditoría completa — valida todas las secciones

/guardian SLUG

Validar una sola sección por su slug

/guardian SLUG1 SLUG2

Validar varias secciones a la vez

/guardian --fix SLUG

Validar Y aplicar correcciones (pide confirmación)

# Auditoría completa
/guardian

# Una sola sección
/guardian hooks

# Varias secciones
/guardian models tokens mcp

# Validar y corregir
/guardian --fix hooks

Cómo funciona — 4 pasos

  1. 1
    Descarga la página relevante

    Obtiene la sección de docs.anthropic.com

  2. 2
    Lee tus archivos de sección

    Lee tanto en/sections/SLUG.html como es/sections/SLUG.html

  3. 3
    Compara cada dato factual

    Flags CLI, nombres de modelos, context windows, nombres de eventos, config keys

  4. 4
    Genera un reporte

    Tres categorías: ✅ Correcto · ⚠️ Desactualizado · ❌ Faltante

Categorías del reporte

✅
Correcto

Coincide exactamente con la documentación oficial

⚠️
Desactualizado / Inexacto

Existe pero no coincide con la versión oficial actual

❌
Faltante

Está en los docs oficiales pero ausente en tu sección

Ejemplo de reporte generado

## Guardian Report: hooks
Validated: 2026-03-28
Official source: https://docs.anthropic.com/en/docs/claude-code/hooks

### ✅ Correcto
- PostToolUse event name — matches official docs
- Hook timeout default (60s) — correct

### ⚠️ Desactualizado / Inexacto
- "stdin" payload field listed as "input" → official docs use "stdin"
  File: en/sections/hooks.html (look for: "input")

### ❌ Faltante
- PreCompact hook event not documented (added in recent release)

| Section | Status |  Issues                 |
|---------|--------|-------------------------|
| hooks   | ⚠️     | 1 outdated, 1 missing   |

Agente Content Writer

Crear / Actualizar

¿Qué hace?

Crea nuevas secciones bilingües o actualiza las existentes, siempre basándose en la documentación oficial de Anthropic. Maneja automáticamente los archivos en inglés y español.

Comandos

/content-writer add SLUG

Crear una sección bilingüe nueva desde cero

/content-writer update SLUG

Reescribir una sección desde los docs oficiales actuales

/content-writer patch SLUG --add "X"

Insertar un dato específico en una sección

/content-writer translate SLUG

Sincronizar la versión en español con la inglesa (o viceversa)

# Agregar una sección completamente nueva
/content-writer add rate-limits

# Reescribir una sección existente
/content-writer update hooks

# Agregar un dato puntual
/content-writer patch hooks --add "PreCompact event"

# Sincronizar español ↔ inglés
/content-writer translate hooks

¿Qué hace cada subcomando?

ComandoAcciónResultado
add SLUG Sección nueva Crea en/sections/SLUG.html + es/sections/SLUG.html, actualiza la navegación en ambos index.html
update SLUG Reescritura completa Muestra diferencias vs docs oficial, pide confirmación y reescribe EN + ES
patch SLUG --add "X" Adición puntual Inserta el contenido en el lugar correcto dentro de ambas versiones
translate SLUG Sincronizar idiomas Identifica qué falta en cada idioma y traduce el contenido diferente

ElementoConvenciónEjemplo

Flujos de Trabajo Combinados

End-to-end

1. Agregar una nueva sección (tras un release de Anthropic)

# Iniciar sesión
cd claude-summary && claude

# Crear la nueva sección
/content-writer add memory-management

# Validar inmediatamente
/guardian memory-management

# Aplicar correcciones si las hay
/guardian --fix memory-management

# Salir y commitear
/exit
git add en/sections/memory-management.html es/sections/memory-management.html \
        en/index.html es/index.html
git commit -m "feat: add memory-management section"

2. Corregir contenido desactualizado en una sección existente

claude
/content-writer update hooks
/guardian --fix hooks
/exit
git add en/sections/hooks.html es/sections/hooks.html
git commit -m "fix(hooks): update to match current official docs"

3. Agregar un dato puntual (patch)

claude
/content-writer patch hooks --add "PreCompact event"
/guardian --fix hooks
/exit
git add en/sections/hooks.html es/sections/hooks.html
git commit -m "fix(hooks): add PreCompact event documentation"

4. Sincronizar traducciones

claude
/content-writer translate hooks
/guardian hooks   # opcional: verificar la versión sincronizada
/exit
git add es/sections/hooks.html
git commit -m "fix(hooks): sync Spanish translation with English"

5. Guardian corrige solo (sin necesitar Content Writer)

claude
/guardian --fix models
/exit
# Guardian muestra los archivos exactos a stagear
git commit -m "fix(models): correct outdated model names and context windows"

Solución de Problemas

Problemas comunes
Las skills no aparecen en /skills
Cierra la sesión con /exit y ábrela de nuevo — las skills solo se cargan al inicio de la sesión.
WebFetch falla para docs.anthropic.com
El sitio puede estar temporalmente no disponible. Espera unos minutos. Guardian intentará automáticamente un fallback con WebSearch.
Content Writer genera HTML malformado
Ejecuta Guardian inmediatamente después: /guardian SLUG. Identificará problemas estructurales. También puedes validarlo manualmente con las dev tools del navegador (F12 → Consola).
El agente edita el archivo incorrecto
Verifica que el slug que pasaste coincide exactamente con el nombre de archivo. Los slugs son case-sensitive en sistemas Linux/Mac.

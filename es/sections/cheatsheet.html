📌
Comandos reales de la CLI de Claude Code — los que aparecen al tipear / o al iniciar una sesión en la terminal.

🚀 Comandos de inicio

claudeInicia sesión interactiva en el directorio actual
claude "tarea"Ejecuta tarea en modo no interactivo
claude -p "prompt"Modo headless / print: envía prompt y sale (sin sesión interactiva)
claude -cRetoma la conversación más reciente en el directorio actual (--continue)
claude -r [sesión]Retoma sesión por ID/nombre; sin arg muestra selector interactivo (--resume)
claude --model <id>Especifica el modelo a usar (ej: sonnet, opus)
claude --worktreeInicia en un git worktree aislado (-w)

⚡ Slash commands del sistema

⚠️
No todos los comandos están disponibles en todos los entornos. La disponibilidad depende del plan, la plataforma (CLI, VS Code extension, etc.) y la versión instalada. Usá /help para ver los activos en tu sesión.
/helpLista todos los comandos disponibles en la sesión actual
/statusAbre la pestaña Status de configuración
/clearLimpia el historial de la conversación (aliases: /reset, /new)
/compact [instrucciones]Comprime el historial para liberar espacio de contexto
/initGenera un CLAUDE.md inicial analizando el proyecto actual
/configAbre la interfaz de configuración (alias: /settings)
/permissionsVer y actualizar permisos (alias: /allowed-tools)
/memoryEditar archivos CLAUDE.md y memoria automática
/contextVisualizar el uso del contexto actual
/model [modelo]Seleccionar o cambiar el modelo de IA
/skillsLista todas las skills disponibles para el directorio actual
/mcpGestionar conexiones a servidores MCP
/agentsGestionar configuraciones de agentes
/hooksVer las configuraciones de hooks activas
/doctorDiagnóstico de instalación y configuración
/plan [descripción]Entrar en modo planificación (solo planifica, no ejecuta)
/resume [sesión]Retomar conversación (alias: /continue)
/exitCierra la sesión (alias: /quit)

📋 Flags globales útiles

--allowedToolsHerramientas que corren sin pedido de permiso
--disallowedToolsHerramientas removidas del contexto del modelo
--output-formatFormato de salida: text, json, stream-json (modo print)
--max-turns <n>Límite de turnos agénticos (modo print)
--verboseLogging detallado turno a turno
--permission-modeInicia en modo específico (ej: plan)
--dangerously-skip-permissions⚠️ Omite todas las confirmaciones — solo entornos automatizados

📁 Configuración

claude config listMuestra la configuración activa
claude config get <key>Valor de una clave específica
claude config set <key> <val>Establece un valor de configuración
claude --versionMuestra la versión instalada

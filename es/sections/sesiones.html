🏷️ Sesiones Flujo de trabajo↗ Docs

💡
Idea clave: Una sesión = una tarea. Mantener sesiones separadas por objetivo mantiene el contexto limpio, reduce el costo por turno y permite retomar el trabajo exactamente donde lo dejaste.

🔄 Comandos de sesión

ComandoQué haceCuándo usarlo
claudeInicia una sesión nueva con contexto limpioCada vez que empezás una tarea nueva
claude -c
claude --continue		Retoma la última sesión activa desde donde quedóQuerés seguir con lo que estabas haciendo
claude -r <session-id>
claude --resume <session-id>		Retoma una sesión específica por su IDQuerés volver a una tarea anterior, no la última
claude -n <nombre>
claude --session-name <nombre>		Inicia una sesión nueva con un nombre personalizadoQuerés identificar la sesión fácilmente sin depender del ID autogenerado

🏷️ Cómo identificar y retomar sesiones

Cada sesión tiene un ID único generado automáticamente. Para poder retomar una sesión específica después:

1

Describí el objetivo al inicio

El primer mensaje de la sesión es el ancla de contexto. Si lo escribís con claridad — qué estás construyendo, qué restricciones hay, qué decidiste — cuando retomés con --resume, Claude recupera el hilo sin que tengas que reexplicar todo.

> "Estoy migrando el módulo de autenticación de JWT a OAuth2.
> Stack: Node.js + Express. Ya completamos el modelo de usuario.
> Falta: middleware de validación de tokens y rutas protegidas."
2

Encontrá el session-id

El ID de sesión aparece en el prompt de Claude Code al iniciar o al listar sesiones. También podés usar:

claude --list-sessions

Esto muestra las sesiones recientes con su ID, fecha y el primer mensaje.

3

Retomá con el ID o el nombre

claude --resume abc123def       # por ID autogenerado
claude --resume mi-tarea        # por nombre asignado con -n

Claude carga el historial completo de esa sesión y continúa como si no hubiera habido interrupción.

🗜️ Sesión larga: usá /compact antes de retomar

Si una sesión tiene muchos turnos acumulados, antes de retomar con --resume conviene comprimirla primero. Así el contexto que se carga es denso y relevante, no el historial crudo completo.

/compact focalizá en el estado actual del módulo de autenticación

✂️ Por qué una sesión por tarea

Una sesión por tareaTodo en una sola sesión
Contexto✅ Limpio y relevante❌ Acumulado con tareas anteriores
Costo por turno✅ Menor (menos tokens de entrada)❌ Crece con cada tarea nueva
Coherencia de Claude✅ Foco en la tarea actual❌ Riesgo de mezclar contexto de tareas distintas
Capacidad de retomar✅ Retomás exactamente esa tarea❌ Difícil saber dónde quedó cada cosa
⚠️
Verificá los flags en tu versión. Los comandos --resume y --list-sessions están documentados en la CLI. Siempre podés ejecutar claude --help para ver todos los flags disponibles en tu versión instalada.

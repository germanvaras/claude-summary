🧩 Skills Conocimiento especializado↗ Docs

💡
Idea clave: Si te encontrás explicándole lo mismo a Claude repetidamente, eso es una skill esperando ser escrita. Una skill se escribe una vez y se aplica sola siempre que es relevante.

📌 ¿Qué son las Skills?

Una skill es un archivo SKILL.md dentro de su propia carpeta, que le enseña a Claude cómo realizar una tarea específica. Claude detecta automáticamente cuándo activarla comparando tu solicitud con la descripción de la skill (semantic matching). No necesitás invocarla manualmente.

🚀 Crear tu primera skill — paso a paso

1

Crear la estructura de carpetas

Cada skill vive en su propia carpeta dentro de .claude/skills/:

# Windows PowerShell
mkdir .claude\skills\pr-review

# Linux / macOS / Git Bash
mkdir -p .claude/skills/pr-review
2

Crear el archivo SKILL.md con frontmatter

# .claude/skills/pr-review/SKILL.md
---
name: pr-review
description: Reviews pull requests for code quality, security and conventions.
             Use when reviewing PRs, checking code before merge, giving feedback
             on a branch, or when asked to review code changes.
---

## Proceso de revisión

1. Leé el diff completo antes de comentar
2. Verificar: convenciones de naming, manejo de errores, tests
3. Indicar severity: [blocking] / [suggestion] / [nitpick]
4. Si hay problemas de seguridad, marcarlos como [critical]
5. Al terminar: resumen con aprobación o lista de cambios requeridos
📌
La description es lo más importante. Claude la usa para decidir cuándo activar la skill. Incluí verbos y frases que los usuarios van a escribir: "reviewing PRs", "checking code before merge", "give feedback on". Máximo 1.024 caracteres.
3

Reiniciar la sesión de Claude Code

Las skills se cargan al iniciar la sesión. Después de crear o modificar una skill, cerrá la sesión y abrí una nueva:

/exit
# luego en la terminal:
claude
4

Verificar que la skill cargó

/skills

Debe aparecer pr-review en la lista con su descripción y origen (personal o proyecto).

Si no aparece: revisá la ubicación del archivo (debe ser .claude/skills/pr-review/SKILL.md, no .claude/skills/SKILL.md directamente) y que el frontmatter tenga los campos name y description.

5

Probar la skill

"revisá el PR de la rama feature/login"

Claude debería activar la skill pr-review automáticamente. Si no la activa, la descripción puede ser demasiado vaga — agregá más frases de ejemplo.

📁 ¿Dónde viven las Skills?

TipoUbicaciónAlcance
Skills de proyecto.claude/skills/nombre/SKILL.md (en la raíz del repo)Compartidas con todo el equipo al clonar. Se commitean al repo.
Skills personales~/.claude/skills/nombre/SKILL.mdTe siguen en todos tus proyectos. No se commitean.
Windows (personal)C:\Users\<usuario>\.claude\skills\nombre\SKILL.mdEquivalente a personal en Windows.

⚖️ Skills vs. CLAUDE.md vs. Slash Commands

MecanismoCuándo cargaCómo se activaCaso de uso ideal
📄 CLAUDE.mdEn cada conversaciónAutomático siempreReglas globales del proyecto: stack, convenciones, restricciones
🧩 SkillsA demanda, cuando coincide semánticamenteAutomático por contextoConocimiento especializado para tareas específicas

🔧 Campos del frontmatter

CampoRequeridoDescripción
name✅ SíIdentificador único. Solo minúsculas, números y guiones. Máx. 64 caracteres.
description✅ SíLe dice a Claude cuándo usar la skill. Cuanto más específica y con más ejemplos de frases, mejor. Máx. 1.024 caracteres.
allowed-tools❌ NoRestringe qué herramientas puede usar Claude cuando la skill está activa. Útil para skills de solo lectura.
model❌ NoEspecifica qué modelo usar cuando esta skill está activa. Ej: claude-haiku-4-5-20251001 para skills de tareas simples.

📂 Progressive disclosure con references/

Si tu skill necesita guías extensas, no las pongas todas en SKILL.md — eso ocupa contexto siempre. Usá la carpeta references/:

.claude/skills/refactor-dotnet/
├── SKILL.md          ← instrucciones esenciales (<500 líneas)
└── references/
    ├── patterns.md   ← guías extensas de CQRS, Repository, etc.
    └── examples.md   ← ejemplos de código de referencia

En el SKILL.md, referencialos explícitamente para que Claude los lea cuando los necesita:

---
name: refactor-dotnet
description: Refactoriza código C# / .NET 8...
---

## Instrucciones base
1. Leé el archivo completo antes de modificar
2. Aplicar patrones de la guía en references/patterns.md si aplica
3. Al terminar: `dotnet build` y listar cambios

## Referencia
- Patrones detallados: @references/patterns.md
- Ejemplos: @references/examples.md
📌
Claude solo carga el contenido de references/ cuando lo necesita para la tarea. SKILL.md siempre se carga, references/ se carga on-demand. Esto evita llenar el contexto con material que no siempre es necesario.

🔄 Múltiples skills activas simultáneamente

Claude puede activar más de una skill al mismo tiempo si la tarea coincide con varias. Las instrucciones se combinan en el contexto. Si hay conflictos entre instrucciones de distintas skills, CLAUDE.md tiene precedencia, y entre skills gana la de mayor prioridad en el orden de la tabla de arriba.

⚠️
Si tenés skills con instrucciones contradictorias (ej: una dice "usá tabs" y otra "usá espacios"), el resultado puede ser inconsistente. Diseñá las skills para que sean complementarias, no solapadas.

🔌 MCP Servers Herramientas externas↗ Docs

💡
Idea clave: MCP (Model Context Protocol) es el sistema que permite a Claude interactuar con herramientas externas reales: bases de datos, APIs, servicios cloud. Sin MCP, Claude solo trabaja con archivos locales. Con MCP, puede hacer queries SQL, crear issues en GitHub, buscar en la web y mucho más.

🧩 Qué es MCP realmente

MCP es un protocolo estándar abierto (publicado por Anthropic, con spec pública) que define cómo los modelos de lenguaje se comunican con herramientas externas. La idea clave es la estandarización: un servidor MCP funciona con cualquier cliente compatible (Claude Code, Claude Desktop, otros).

Un servidor MCP es un proceso separado que expone capacidades al modelo. Claude Code lo inicia, se conecta a él y puede invocar sus funciones durante una tarea. Desde el punto de vista de Claude, las herramientas del servidor MCP aparecen como herramientas adicionales junto a Read, Write, Bash, etc.

Claude Code (cliente MCP)
│
├── Herramientas nativas: Read, Write, Edit, Bash, Grep...
│
└── Herramientas MCP (vía servidor externo):
    ├── query_database(sql)         ← postgres MCP server
    ├── create_issue(title, body)   ← github MCP server
    └── search_web(query)           ← brave-search MCP server

🔌 Tipos de transporte

Los servidores MCP pueden comunicarse con Claude Code de dos formas:

TransporteCómo funcionaCuándo usarloConfig
stdio Claude Code lanza el server como proceso hijo. Comunican por stdin/stdout (JSON-RPC). El server vive mientras Claude Code corre. La mayoría de los casos: servers locales, npx, binarios command + args
HTTP (Streamable HTTP) El server corre independientemente (puede ser remoto). Claude Code se conecta vía HTTP POST/GET. Opcionalmente usa SSE para streaming de respuestas. Servers remotos, servicios cloud, APIs compartidas del equipo. Recomendado para casos remotos. type: "http" + url + headers
SSE ⚠️ Deprecado Transporte anterior basado en Server-Sent Events. Sigue funcionando pero está deprecado. Solo para compatibilidad con servidores legados que aún no migraron a HTTP. type: "sse" + url
📌
Para servidores locales (npx, binarios), usá stdio. Para servicios remotos o APIs del equipo, usá HTTP — es el transporte recomendado para uso cloud. Evitá SSE en servidores nuevos: está deprecado.

📦 Instalar un servidor MCP

Hay tres formas de instalar un servidor:

A

Via npx (automático, sin instalación previa)

El método más simple: ponés npx como comando y Claude Code descarga el servidor la primera vez que lo necesita.

"command": "npx",
"args": ["-y", "@modelcontextprotocol/server-github"]

El -y acepta automáticamente la instalación. Requiere conexión a internet y Node.js.

⚠️
En redes corporativas con proxy, npx puede fallar. En ese caso, instalá el paquete globalmente primero: npm install -g @modelcontextprotocol/server-github y luego usá el binario directamente.
B

Instalación global con npm

npm install -g @modelcontextprotocol/server-postgres

Más confiable para uso continuo. Después de instalar, podés usar el comando directamente en la config sin npx:

"command": "mcp-server-postgres",
"args": ["postgresql://user:pass@localhost/mibase"]
C

Servidor HTTP remoto

Si el servidor ya corre en algún lugar (tu infra, un servicio del equipo):

"url": "http://mcp-server.interno:8080"

⚙️ Configuración en settings.json

Los servidores MCP se configuran en el bloque mcpServers de settings.json. Podés tener múltiples servidores activos simultáneamente.

Estructura para servidor stdio:

{
  "mcpServers": {
    "nombre-del-server": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-NOMBRE"],
      "env": {
        "VARIABLE": "valor"
      }
    }
  }
}

Estructura para servidor HTTP:

{
  "mcpServers": {
    "mi-server-remoto": {
      "type": "http",
      "url": "https://api.ejemplo.com/mcp",
      "headers": {
        "Authorization": "Bearer ${API_KEY}"
      }
    }
  }
}
⚠️
No pongas credenciales reales en .claude/settings.json del proyecto (va al repositorio). Usá settings.local.json para valores sensibles, o variables de entorno del sistema en lugar del bloque env.

🧱 Los tres componentes de MCP

Un servidor MCP puede exponer hasta tres tipos de capacidades:

ComponenteQué esCómo lo usa ClaudeEjemplo
Tools Funciones que Claude puede invocar para hacer algo: ejecutar queries, crear recursos, llamar APIs. Automáticamente cuando son relevantes, o cuando vos lo pedís explícitamente. Aparecen en el contexto igual que Read o Bash. execute_query(sql), create_issue(title, body), post_message(channel, text)
Resources Datos que Claude puede leer: registros de base de datos, archivos remotos, esquemas. Son read-only por definición. Claude los solicita para obtener contexto. El servidor devuelve el contenido que se inyecta al contexto de Claude. Esquema completo de la DB, documentación de la API, logs de la última hora
Prompts Templates de prompts predefinidos por el servidor que aparecen como slash commands con el prefijo /mcp__servidor__nombre. El usuario los invoca manualmente, igual que un slash command custom. /mcp__github__create-pr, /mcp__postgres__explain-query

🔌 Servers verificados y disponibles

El namespace oficial en npm es @modelcontextprotocol/. Antes de usar cualquier servidor de terceros, verificá que exista en npmjs.com.

PaqueteQué proveeCredencial requerida
@modelcontextprotocol/server-postgresQueries y esquema de PostgreSQLConnection string en args
@modelcontextprotocol/server-githubRepos, issues, PRs, búsqueda de códigoGITHUB_PERSONAL_ACCESS_TOKEN
@modelcontextprotocol/server-slackLeer canales, enviar mensajesSLACK_BOT_TOKEN, SLACK_TEAM_ID
@modelcontextprotocol/server-brave-searchBúsqueda web via Brave Search APIBRAVE_API_KEY
@modelcontextprotocol/server-sqliteQueries SQLite sobre archivos localesRuta al archivo .db en args
@modelcontextprotocol/server-filesystemAcceso extendido al sistema de archivosRutas permitidas en args
📌
No existe un servidor oficial para SQL Server (MSSQL) en el namespace @modelcontextprotocol/. Para SQL Server, buscá en npmjs.com o considerá exponer los datos via un servidor propio con el SDK de MCP.

💡 Ejemplo funcional completo — GitHub

Configuración en .claude/settings.local.json (no va al repo — tiene el token):

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_tu_token_aqui"
      }
    }
  }
}

Una vez conectado, Claude puede hacer esto automáticamente:

> "buscá issues abiertos con el label 'bug' en el repo actual"
# Claude invoca: list_issues(labels=["bug"], state="open")
# Muestra: 3 issues encontrados con su título, número y descripción

> "creá un issue: 'Fix null reference en OrderService' con label bug"
# Claude invoca: create_issue(title=..., body=..., labels=["bug"])
# Resultado: Issue #47 creado en GitHub

> "mostrá el último PR mergeado"
# Claude invoca: list_pull_requests(state="closed", sort="updated")
# Muestra: PR #46 - "Add CQRS to Orders module"

💡 Ejemplo funcional completo — PostgreSQL

Configuración en .claude/settings.local.json:

{
  "mcpServers": {
    "staging-db": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-postgres",
        "postgresql://readonly_user:pass@staging-db:5432/ecommerce"
      ]
    }
  }
}

En la sesión:

> "¿cuántos pedidos tiene el estado 'pending' de los últimos 7 días?"
# Claude invoca: query("SELECT COUNT(*) FROM orders WHERE status='pending' AND created_at > NOW()-INTERVAL '7 days'")
# Resultado: 142 pedidos

> "mostrá el esquema de la tabla orders"
# Claude invoca: describe_table("orders")
# Muestra: columnas, tipos, constraints, índices

🔧 Cómo debuggear la conexión MCP

1

Verificar estado dentro de la sesión

/mcp

Muestra todos los servidores configurados con su estado: conectado, desconectado o error.

2

Ver errores de inicialización

claude --verbose

En modo verbose verás exactamente por qué falla la conexión: paquete no instalado, credencial incorrecta, host no accesible.

3

Checklist de problemas comunes

SíntomaCausa probableSolución
Server aparece como desconectadoPaquete npx no instalado / red bloqueadaInstalar globalmente: npm install -g @modelcontextprotocol/server-NOMBRE
Error de autenticaciónVariable de entorno faltante o incorrectaVerificar que la variable esté definida: echo %GITHUB_PERSONAL_ACCESS_TOKEN%
Server conectado pero sin tools disponiblesError al inicializar el servidorCorrer claude --verbose y leer el stack trace del servidor
Connection refused (HTTP server)El servidor remoto no está corriendoVerificar con curl http://host:puerto/health

🏗️ Crear tu propio servidor MCP

Si necesitás integrar una API o sistema que no tiene servidor MCP disponible, podés crear el tuyo usando el SDK oficial:

SDKPaquete
TypeScript / JavaScript@modelcontextprotocol/sdk
Pythonmcp (PyPI)

La spec completa está en modelcontextprotocol.io. El SDK maneja el protocolo; vos solo definís qué tools, resources y prompts querés exponer.

📌
Un servidor MCP propio es la solución correcta cuando necesitás conectar Claude con sistemas internos: tu ERP, APIs privadas, bases de datos legacy o cualquier servicio que no tiene un servidor MCP público.

Términos del ecosistema Claude Code — todo lo que necesitás entender para leer esta documentación sin googlear cada dos líneas.
Context Window
La cantidad máxima de tokens que Claude puede procesar en una interacción. Este presupuesto cubre input + output combinados: historial, CLAUDE.md, skills, archivos leídos (input) y la respuesta generada (output). Los modelos Opus 4.6, Sonnet 4.6 y Haiku 4.5 tienen 200.000 tokens. El máximo de output por respuesta es un límite separado, sustancialmente menor.
Si el contexto se llena, Claude pierde el foco en instrucciones del inicio de la conversación — no es un bug, es el límite del context window
Token
Unidad básica con la que los modelos de lenguaje procesan texto. No es una palabra ni un carácter — es un fragmento que puede ser parte de una palabra, una palabra entera o un signo de puntuación. Aproximadamente 3–4 caracteres en inglés. En español el tokenizer es menos eficiente — el mismo texto consume más tokens que en inglés.
Un CLAUDE.md de 500 palabras consume ~700 tokens del contexto en cada conversación
Frontmatter
Bloque de metadatos al inicio de un archivo Markdown, delimitado por ---. En Claude Code se usa para definir el name, description y campos opcionales de skills y slash commands personalizados.
El bloque --- name: pr-review --- al inicio de un SKILL.md es el frontmatter
Semantic Matching
El mecanismo por el cual Claude compara tu solicitud con el campo description de las skills disponibles para decidir cuál activar. No busca palabras exactas — interpreta la intención y el significado de lo que pedís. Por eso la descripción de la skill es tan importante.
"revisá el PR" puede activar una skill descrita como "reviews code changes before merge"
Skill
Archivo SKILL.md dentro de su propia carpeta que le enseña a Claude cómo realizar una tarea específica. Se activa automáticamente cuando Claude detecta que tu solicitud coincide semánticamente con su descripción. No requiere invocación manual.
Una skill de pr-review se activa sola cuando pedís revisar un PR
Agente
Instancia de Claude que opera con autonomía dentro de un contexto de ejecución: puede invocar herramientas, leer y escribir archivos, ejecutar comandos y encadenar acciones para completar una tarea compleja sin intervención continua.
Claude refactorizando un módulo entero por su cuenta, archivo por archivo, verificando que compile
Subagente
Instancia de Claude lanzada por el agente principal para ejecutar una tarea delegada en un contexto completamente aislado. Recibe instrucciones, trabaja de forma independiente y devuelve resultados sin "ver" la conversación principal.
El agente principal delega "escribir los tests" a un subagente mientras continúa con otra tarea
Execution Context
El entorno aislado en el que opera un agente o subagente: qué herramientas tiene disponibles, qué instrucciones recibió y cuál es el estado de su conversación. Cada subagente tiene el suyo propio, separado del principal.
El subagente de tests no "ve" el historial de conversación del agente principal
MCP (Model Context Protocol)
Protocolo abierto estándar que define cómo los modelos de lenguaje se comunican con herramientas y servicios externos. Permite a Claude conectarse a bases de datos, APIs y servicios sin código de integración personalizado. El namespace oficial de servidores en npm es @modelcontextprotocol/.
Un MCP server de PostgreSQL permite a Claude ejecutar SQL queries directamente contra una base de datos
MCP Server
Proceso externo que implementa el protocolo MCP y expone capacidades (Tools, Resources, Prompts) que Claude puede usar durante una tarea. Se configura en settings.json y corre como proceso separado al iniciar Claude Code. Puede usar transporte stdio (proceso local) o HTTP (servidor remoto).
@modelcontextprotocol/server-github expone la API de GitHub como tools que Claude puede invocar
Hook
Script o comando que Claude Code ejecuta automáticamente cuando ocurre un evento específico — sin que el usuario lo pida. Los hooks reciben información del evento via stdin (JSON) y pueden permitir o bloquear acciones mediante su exit code.
Un hook PostToolUse corre el linter cada vez que Claude guarda un archivo
Matcher
Expresión regular (regex) en la configuración de hooks que define contra qué herramienta o evento se compara. El carácter | es el operador OR de regex. Omitir el campo hace que el hook dispare ante cualquier herramienta.
"matcher": "Write|Edit" activa el hook cuando Claude escribe o edita archivos, pero no en Bash o Read
Slash Command
Instrucción predefinida que el usuario invoca manualmente escribiendo /nombre en la conversación. Existen tres tipos: built-in (del sistema), skill commands (creados por vos en .claude/commands/), y MCP prompts (expuestos por servidores MCP).
/coverage-report genera el reporte de cobertura cuando el desarrollador lo decide
$ARGUMENTS
Variable especial disponible en archivos de slash commands personalizados. Captura todo el texto que el usuario escribe después del nombre del comando. Permite crear comandos parametrizados.
En /review feature/login, $ARGUMENTS captura el valor feature/login
Progressive Disclosure
Técnica de organización donde solo se carga el contenido esencial inicialmente, y el detalle adicional se incorpora bajo demanda. En skills: SKILL.md tiene lo esencial, la carpeta references/ tiene el material extenso que solo carga cuando se necesita.
La guía de arquitectura de 200 líneas solo se carga al contexto cuando alguien pregunta sobre arquitectura
allowed-tools
Campo opcional en el frontmatter de una skill que restringe qué herramientas nativas puede usar Claude mientras esa skill está activa. Útil para crear skills de solo lectura o con guardarraíles de seguridad.
allowed-tools: Read, Grep impide que Claude escriba archivos mientras la skill está activa
Settings Cascade
El sistema de resolución de configuración de Claude Code, donde múltiples archivos settings.json se combinan según su nivel. El orden de precedencia (de mayor a menor) es: Managed → Local → Project → User. Las reglas del nivel más específico prevalecen.
Una regla en .claude/settings.local.json (Local) sobreescribe la configuración de .claude/settings.json (Project)
Worktree
Un directorio de trabajo adicional vinculado al mismo repositorio Git, que permite tener múltiples ramas activas simultáneamente. Claude Code usa el flag --worktree para crear un worktree aislado donde el agente trabaja sin afectar tu directorio principal.
claude --worktree crea una copia aislada del repo para que Claude haga refactoring sin tocar tu trabajo actual
Headless Mode / Print Mode
Modo de ejecución no interactivo de Claude Code, activado con claude -p "tarea". Envía el prompt, espera la respuesta completa y sale. No abre una sesión interactiva. Diseñado para uso en scripts, pipelines de CI/CD y automatizaciones.
claude -p "generá el changelog de la versión 2.4.0" --output-format json retorna el resultado como JSON y sale
/doctor
Slash command de diagnóstico de Claude Code que verifica el estado del entorno: versión instalada, autenticación, conectividad con la API y configuración activa. Primer paso ante cualquier problema técnico.
Si Claude Code falla silenciosamente, abrí una sesión y ejecutá /doctor para ver el diagnóstico

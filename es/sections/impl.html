📄 CLAUDE.md — contexto permanente

# EcommerceApi

## Stack
- .NET 8 · ASP.NET Core Minimal APIs
- Entity Framework Core 8 · SQL Server 2022
- xUnit · Moq · FluentAssertions

## Convenciones
- Usar record types para DTOs y Value Objects
- Métodos async terminan en Async
- Nunca retornar null: usar Result<T> (FluentResults)
- EF Core: AsNoTracking() en queries de solo lectura
- Patrón CQRS: Commands separan de Queries

## Restricciones
- Nunca modificar migraciones existentes — crear nueva migración
- No usar DbContext en Controllers — solo en Repositories
- No hardcodear connection strings

## Comandos
- `dotnet build`
- `dotnet test --filter Category=Unit`
- `dotnet ef migrations add <Nombre>`
- `dotnet format`

🪝 Hook: build automático tras cada cambio

En .claude/settings.json (va al repo — el equipo lo comparte):

{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{
        "type": "command",
        "command": "dotnet format --verify-no-changes 2>&1 | tail -3 && dotnet build --no-restore -q 2>&1 | tail -5"
      }]
    }]
  }
}
⚠️
Asegurate de que dotnet esté en el PATH del sistema, no solo del shell interactivo. Si el hook falla con "command not found", usá la ruta absoluta al ejecutable de dotnet.

🔌 MCP + ⚡ /coverage-report

En .claude/settings.local.json (NO va al repo — tiene la connection string real):

{
  "mcpServers": {
    "staging-db": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": { "POSTGRES_URL": "postgresql://readonly:pass@staging-db:5432/ecommerce" }
    }
  }
}

En .claude/commands/coverage-report.md:

Ejecutá: dotnet test --collect:"XPlat Code Coverage"
Analizá el reporte XML generado y reportá:
- Clases bajo el $ARGUMENTS% de cobertura
- Métodos más críticos sin test
- Sugerencia de casos de test para los 3 más importantes

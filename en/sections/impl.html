📄 CLAUDE.md — permanent context

# EcommerceApi

## Stack
- .NET 8 · ASP.NET Core Minimal APIs
- Entity Framework Core 8 · SQL Server 2022
- xUnit · Moq · FluentAssertions

## Conventions
- Use record types for DTOs and Value Objects
- Async methods end in Async
- Never return null: use Result<T> (FluentResults)
- EF Core: AsNoTracking() on read-only queries
- CQRS pattern: separate Commands from Queries

## Restrictions
- Never modify existing migrations — create a new migration
- Do not use DbContext in Controllers — only in Repositories
- Do not hardcode connection strings

## Commands
- `dotnet build`
- `dotnet test --filter Category=Unit`
- `dotnet ef migrations add <Name>`
- `dotnet format`

🧩 Skill: refactor-dotnet

---
name: refactor-dotnet
description: Refactors C# / .NET 8 code. Use when asked
             to refactor, improve or modernize classes, services
             or C# repositories. Also when migrating to CQRS or Result<T>.
---
1. Read the full file before modifying
2. Apply patterns described in references/patterns.md when applicable
3. EF Core: AsNoTracking() on queries, transactions on writes
4. When done: `dotnet build` and list changes made
5. Indicate which new tests are needed

🧩 Skill: test-writer

---
name: test-writer
description: Writes .NET unit and integration tests.
             Use when asked to create tests, add coverage
             or test a specific method, class or endpoint.
---
- Framework: xUnit · Moq (MockBehavior.Strict) · FluentAssertions
- Naming: Method_Scenario_Result
- Structure: Arrange / Act / Assert always
- Cover: happy path, invalid input, not found, infra error

🪝 Hook: automatic build after each change

In .claude/settings.json (goes to repo — the team shares it):

{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{
        "type": "command",
        "command": "dotnet format --verify-no-changes 2>&1 | tail -3 && dotnet build --no-restore -q 2>&1 | tail -5"
      }]
    }]
  }
}
⚠️
Make sure dotnet is in the system PATH, not just the interactive shell's PATH. If the hook fails with "command not found", use the absolute path to the dotnet executable.

🔌 MCP + ⚡ /coverage-report

In .claude/settings.local.json (does NOT go to repo — has the real connection string):

{
  "mcpServers": {
    "staging-db": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": { "POSTGRES_URL": "postgresql://readonly:pass@staging-db:5432/ecommerce" }
    }
  }
}

In .claude/commands/coverage-report.md:

Run: dotnet test --collect:"XPlat Code Coverage"
Analyze the generated XML report and report:
- Classes below $ARGUMENTS% coverage
- Most critical untested methods
- Test case suggestions for the 3 most important ones

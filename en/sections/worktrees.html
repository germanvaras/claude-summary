🌿 Worktrees Isolation↗ Docs

💡
Key concept: A worktree is an isolated working copy of the same Git repository. Claude Code can work in a worktree without touching your main directory — ideal for aggressive refactoring, experiments, or high-impact agentic work.

🌳 What is a git worktree?

Git allows you to have multiple working directories linked to the same repository. Each worktree has its own branch and file state, but they share history. It is like having the project "cloned" in another folder, but much more efficient.

# Without worktrees — Claude works directly in your directory
cd C:\projects\my-api
claude
> "refactor the entire Orders module"
# Claude modifies files in your working directory

# With worktrees — Claude works in an isolated copy
cd C:\projects\my-api
claude --worktree
> "refactor the entire Orders module"
# Claude works in a separate copy. Your directory is unchanged.

🚀 How to use worktrees with Claude Code

1

Start Claude in an isolated worktree

claude --worktree

Claude Code automatically creates a new worktree from the current branch, starts the session in that worktree, and works there.

2

Claude works in the worktree

From your main directory you can keep working normally — Claude's changes are in the isolated copy. When the session ends, Claude Code tells you which files changed in the worktree.

3

Review and decide what to do with the changes

# View changes in the worktree
git diff HEAD..worktree/branch

# If you like the changes — merge or create PR
git merge worktree/branch

# If you don't like the changes — discard the worktree
git worktree remove /path/to/worktree

🎯 When to use worktrees

ScenarioWhy worktree
Large, risky refactoringIf Claude makes a mistake, your main branch is not affected. You can discard the worktree and start over.
Experimenting with architectureTry an approach without committing your work in progress.
Long-running autonomous agentThe agent works in parallel in the worktree while you continue on the main branch.
CI/CD with human reviewClaude generates changes in a separate branch via worktree; you review and approve before merging.

⚖️ With worktree vs. without worktree

Without worktreeWith --worktree
Where does Claude work?Directly in your directoryIn an isolated copy
Affects your current work?Yes — modifies the files you have openNo — your directory remains intact
Can you discard everything?Only with git reset or stashYes — simply remove the worktree
Can you review before accepting?You need to revert manuallyYes — see the worktree diff and decide
OverheadNoneUses extra disk space (a copy of the files)
📌
Worktrees require the project to be under Git version control. If the project has no Git repository, the --worktree flag cannot create an isolated copy.

🛠️ Useful Git commands with worktrees

# View all active worktrees
git worktree list

# Remove a worktree (when you no longer need it)
git worktree remove /path/to/worktree

# Clean up orphaned worktrees
git worktree prune

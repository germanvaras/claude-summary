🧠 Mental Model Where to start↗ Docs

💡
Before diving into the details: understand what each piece does. Once this is clear, the rest of the documentation will fall into place.

🗺️ The complete map

PieceRoleWhen it applies
📄 CLAUDE.mdPersistent project memoryAlways — loads automatically at the start of every conversation
🧩 SkillsActivatable specialized knowledgeWhen the task semantically matches the skill's description
🪝 HooksEvent-driven automationsWhen Claude does something: writes a file, runs a command, finishes
🔌 MCP ServersConnection to external toolsWhen you need APIs, databases, or third-party services
🔐 PermissionsSecurity guardrailsDefines what Claude can and cannot do in each context
⚡ Slash CommandsExplicit manual actionsWhen you decide to run something, with full control over timing

🔗 How they relate

Each conversation with Claude Code
│
├── Loaded automatically on start
│   ├── CLAUDE.md          → permanent project context
│   └── Relevant skills    → specialized knowledge per task
│
├── Triggered automatically (event-driven)
│   └── Hooks              → react to what Claude does
│
├── You activate manually
│   └── Slash Commands     → /deploy, /changelog, /standup
│
├── Available as tools
│   └── MCP Servers        → APIs, DBs, external services
│
└── Always-active guardrails
    └── Permissions        → what Claude can and cannot do

🎯 Quick decision rule

NeedSolution
Claude knows nothing about the project📄 Write a CLAUDE.md with context
Claude doesn't know how to do something🧩 Create a Skill with the instructions
You want something to happen automatically🪝 Configure a Hook for that event
You need to connect to an API or DB🔌 Add an MCP Server
You want to run something on demand⚡ Create a Slash Command
Claude does things it shouldn't🔐 Configure Permissions in settings.json
✅
These pieces are complementary, not alternatives. A well-configured project uses all of them: CLAUDE.md provides context, skills provide knowledge, hooks provide automation, MCP provides external connections, and permissions provide security.

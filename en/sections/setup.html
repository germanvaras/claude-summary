🚀 Installation

1

Method A — Native installer (recommended for Windows)

Open PowerShell and run:

irm https://claude.ai/install.ps1 | iex

The native installer includes its own dependencies — you do not need to install Node.js separately. It updates automatically in the background.

Alternative via WinGet:

winget install Anthropic.ClaudeCode
⚠️
If PowerShell blocks script execution, run first: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser, then run the installer again.
2

Method B — npm (if you already have Node.js 18+)

npm install -g @anthropic-ai/claude-code

With this method, claude installs to %APPDATA%\npm. If the command is not recognized after installing, verify that path is in the system PATH.

3

Verify the installation

claude --version

Or from within a session, use the slash command:

/doctor

Checks version, permissions, and API connectivity. Reports any issues in detail.

4

First run and authentication

Navigate to any project and run:

claude

The first time, it will prompt for authentication. See the 🔑 Authentication section for available options (browser login or API key).

🖥️ Running from anywhere on Windows

Claude Code uses the directory where you run it as the project root. Always navigate to the right folder before running claude.

TerminalHow to open in the project directory
VS CodeIntegrated terminal (Ctrl+`) — opens directly in the workspace. Just type claude.
Visual StudioMenu View → Terminal (VS 2022+) — opens in the solution directory. Type claude there.
Windows Terminal / PowerShellcd C:\path\to\project then claude.
Windows ExplorerClick the address bar of the project folder → type powershell → Enter. Opens PowerShell directly in that directory.
⚠️
If claude is not recognized as a command after installing: close and reopen the terminal. Windows does not reload PATH automatically in already-open sessions.

🔧 Common issues on Windows

claude not found (npm install)PATH does not include the global npm folderClose and reopen the terminal. If it persists, verify %APPDATA%\npm is in the system PATH.
claude not found (native installer)PATH not reloaded in the current sessionClose and reopen the terminal. If it persists, reinstall using the official script.

Fails on corporate networksProxy blocking the script or Anthropic APIUse WinGet: winget install Anthropic.ClaudeCode.

📁 Verify Claude detected your project

Once inside a session, these commands confirm Claude loaded the correct context:

/statusShows configuration state: active directory, model, loaded skills
/skillsLists available skills for the current project
/mcpStatus of configured MCP servers
/doctorFull environment diagnostic (version, permissions, connectivity)
✅
If /skills lists the project skills and /doctor reports no errors, Claude is ready to work.

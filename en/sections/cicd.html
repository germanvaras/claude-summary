🔄 Claude Code in CI/CD Automation↗ Docs

💡
Key concept: Claude Code has a headless (non-interactive) mode designed for CI/CD pipelines and automation. It runs, completes the task, and exits — no interactive session, with structured output.

🚀 Headless mode — claude -p

The -p flag (print / headless) sends a prompt, waits for the full response, and exits. It does not open an interactive session.

# Basic syntax
claude -p "task you want to execute"

# Example: auto-generate changelog
claude -p "generate the changelog for version 2.4.0 based on commits since the v2.3.0 tag"

# Example: review code before merge
claude -p "review the modified files in this PR and list any critical issues"

📤 Output formats

In headless mode you can control the output format:

FlagFormatWhen to use it
--output-format textPlain text (default)Scripts that read the output as a string
--output-format jsonJSON with the full response at the endIntegration with tools that parse JSON
--output-format stream-jsonJSON events as a stream, one per line (NDJSON)Process the response in real time as it arrives
 
# JSON output for script parsing
claude -p "list classes with coverage below 80%" --output-format json

# Stream for real-time processing
claude -p "refactor OrderService.cs" --output-format stream-json | jq '.content'

🔒 Authentication in CI/CD

In pipelines without a graphical interface, browser login does not work. Always use an API key:

# GitHub Actions — secret configured in the repository
env:
  ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

# Docker
docker run -e ANTHROPIC_API_KEY=sk-ant-... my-image claude -p "task"

# Environment variables in CI system
export ANTHROPIC_API_KEY="sk-ant-api03-..."
claude -p "task"

⚙️ Important CI/CD flags

--max-turns <n>Limits the number of agentic turns. Prevents Claude from entering infinite loops. Recommended: 10–30 depending on complexity.
--dangerously-skip-permissionsBypasses all interactive confirmations. Required in CI (no user to approve). Use with deny rules in settings.json as a guardrail.
--allowedTools Read,Write,BashRestricts which tools Claude can use in this run. Principle of least privilege.
--disallowedTools BashBlocks specific tools for this run.
--output-format jsonJSON output — easier to parse in CI scripts.
--verboseDetailed turn-by-turn logging. Useful for debugging pipelines.

📋 Example — GitHub Actions

# .github/workflows/claude-review.yml
name: Claude Code Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  code-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Claude review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude -p "Review the modified files in this PR. \
            List critical issues, improvement suggestions, and any missing tests. \
            Format: markdown list with severity [critical/suggestion/nitpick]." \
            --max-turns 20 \
            --allowedTools Read,Grep,Glob \
            --output-format text

📋 Example — Automation script

#!/bin/bash
# generate-release-notes.sh

VERSION=$1
PREV_VERSION=$2

echo "Generating release notes for $VERSION..."

NOTES=$(claude -p "Generate release notes for version $VERSION. \
  Based on commits from the $PREV_VERSION tag to HEAD. \
  Group by: Features, Bug Fixes, Breaking Changes. \
  Markdown format." \
  --max-turns 5 \
  --output-format text)

echo "$NOTES" > "RELEASE_NOTES_$VERSION.md"
echo "Done: RELEASE_NOTES_$VERSION.md"

🛡️ Security best practices in CI/CD

PracticeWhy
Use --allowedTools with least privilegeIn CI, Claude doesn't need all tools. Restrict to what the task actually needs.
Configure deny rules in project settings.jsonDeny rules work even with --dangerously-skip-permissions
Never hardcode the API key in code or committed filesUse CI secrets (GitHub Secrets, etc.)
Always use --max-turns in CIPrevents runs that never end and consume credits uncontrolled
Check the process exit codeclaude -p returns 0 on success, non-zero on error — use it to control the pipeline
✅
Ideal combination for CI/CD: ANTHROPIC_API_KEY in secret + --dangerously-skip-permissions + --allowedTools with least privilege + --max-turns as a safety limit + deny rules in project settings.json.

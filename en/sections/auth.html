🔑 Authentication Configuration↗ Docs

💡
Key concept: Claude Code supports two authentication methods: browser login (for interactive use) and API key via environment variable (for CI/CD, servers, or headless environments).

🌐 Method 1 — Browser login (interactive use)

This is the default method. When running claude for the first time, it opens the browser to sign in with your Anthropic account.

1

Run Claude Code

claude

If there is no active session, it displays a message and opens the browser automatically.

2

Approve in the browser

Sign in at claude.ai with your Anthropic account and approve access for Claude Code.

3

Token stored locally

The session token is saved to:

OSLocation
WindowsC:\Users\<user>\.claude\credentials.json
macOS / Linux~/.claude/credentials.json

Subsequent runs reuse this token automatically — no need to log in again.

⚠️
If the token expires or the account changes, run claude logout and log in again. In shared environments (servers, CI), never use browser login — use an API key instead.

🔐 Method 2 — API Key (CI/CD, servers, headless environments)

If you set the ANTHROPIC_API_KEY environment variable, Claude Code uses it automatically without requiring browser login. This method is required in automated environments.

Getting an API key

Go to console.anthropic.com → API Keys → Create Key. Keys begin with sk-ant-api03-...

Setting up on Windows

Current session only:

$env:ANTHROPIC_API_KEY = "sk-ant-api03-..."

Permanent for your user:

setx ANTHROPIC_API_KEY "sk-ant-api03-..."

After setx, close and reopen the terminal for it to take effect.

Verify Claude is using the key

claude --version   # should respond without prompting for login
/doctor            # shows "API key: configured"
📌
If both ANTHROPIC_API_KEY and credentials.json are present, the environment variable takes priority.

🔒 Security best practices

ContextRecommended methodWhy
Local interactive developmentBrowser loginNo credentials in files or variables
CI/CD (GitHub Actions, etc.)Repository secret → env varKey is not exposed in logs or code
Team serverSystem environment variable, managed by ITCentralized control, not in code
Docker / containersEnvironment variable injected at run timedocker run -e ANTHROPIC_API_KEY=... ...
🔴
Never do this:
  • Store the API key in the project's settings.json (it goes to the repository)
  • Hardcode the key in scripts or source code
  • Share ~/.claude/credentials.json

🚀 First use — complete flow from scratch

Once authenticated, this is the typical flow to start working on a project:

# 1. Go to the project directory
cd C:\my-projects\my-api

# 2. Start Claude Code
claude

# 3. If the project has no CLAUDE.md, generate one
/init

# 4. Verify everything loaded correctly
/status
/skills

# 5. Start the first task
"explain the project architecture"
✅
You're now operational. The CLAUDE.md generated by /init gives Claude the project context from the very first message.

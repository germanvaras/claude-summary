📌
Real Claude Code CLI commands — the ones that appear when you type / or when starting a session in the terminal.

🚀 Startup commands

claudeStarts interactive session in the current directory
claude "task"Runs task in non-interactive mode
claude -p "prompt"Headless / print mode: sends prompt and exits (no interactive session)
claude -cResumes the most recent conversation in the current directory (--continue)
claude -r [session]Resumes session by ID/name; without arg shows interactive selector (--resume)
claude --model <id>Specifies the model to use (e.g.: sonnet, opus)
claude --worktreeStarts in an isolated git worktree (-w)

⚡ System slash commands

⚠️
Not all commands are available in all environments. Availability depends on the plan, platform (CLI, VS Code extension, etc.), and installed version. Use /help to see the ones active in your session.
/helpLists all available commands in the current session
/statusOpens the Status configuration tab
/clearClears the conversation history (aliases: /reset, /new)
/compact [instructions]Compresses history to free context space
/initGenerates an initial CLAUDE.md by analyzing the current project
/configOpens the configuration interface (alias: /settings)
/permissionsView and update permissions (alias: /allowed-tools)
/memoryEdit CLAUDE.md files and automatic memory
/contextVisualize current context usage
/model [model]Select or change the AI model
/skillsLists all available skills for the current directory
/mcpManage MCP server connections
/agentsManage agent configurations
/hooksView active hook configurations
/doctorInstallation and configuration diagnostic
/plan [description]Enter planning mode (plans only, does not execute)
/resume [session]Resume conversation (alias: /continue)
/exitClose the session (alias: /quit)

📋 Useful global flags

--allowedToolsTools that run without permission prompts
--disallowedToolsTools removed from the model's context
--output-formatOutput format: text, json, stream-json (print mode)
--max-turns <n>Agentic turn limit (print mode)
--verboseDetailed turn-by-turn logging
--permission-modeStart in a specific mode (e.g.: plan)
--dangerously-skip-permissions⚠️ Bypasses all confirmations — automated environments only

📁 Configuration

claude config listShows the active configuration
claude config get <key>Value of a specific key
claude config set <key> <val>Sets a configuration value
claude --versionShows the installed version

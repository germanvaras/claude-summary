⚙️ How to change the model

The default model in Claude Code is Sonnet. You can change it in several ways:

MethodCommandScope
During the session/model claude-opus-4-6That session only
At startupclaude --model claude-opus-4-6That session only
In settings.json"model": "claude-sonnet-4-6"Permanent (User or Project scope)
In skill frontmattermodel: claude-haiku-4-5-20251001Only when that skill is active
📌
Exact model IDs: claude-opus-4-6 · claude-sonnet-4-6 · claude-haiku-4-5-20251001. Use the full ID in configuration to avoid ambiguity.
⚠️
Haiku is version 4.5, not 4.6 — they are not all from the same generation. The date suffix in claude-haiku-4-5-20251001 is part of the official ID and is required. Using claude-haiku-4-5 without the date or attempting claude-haiku-4-6 may silently fail or fall back to the default model.

💰 Tokens and cost in practice

In Claude Code, cost is measured in tokens consumed — both input (history, files, skills, CLAUDE.md) and output (what Claude generates). The more complex the task and the more accumulated context, the more tokens each turn consumes.

📌
Input and output tokens are not priced the same. Output tokens cost significantly more per token than input tokens. In pipelines where Claude generates long responses (reports, extensive code, analyses), the real cost is dominated by output — not input context. Optimize response size, not just context.
ScenarioIdeal modelWhy
Interactive development sessionSonnetMany turns → accumulated cost matters
One critical focused task (architecture, security)OpusFew turns → extra cost is justified
Autonomous agent in CI/CD pipelineSonnetMany autonomous turns with minimal intervention
Repetitive tasks: formatting, renames, boilerplateHaikuNo need for maximum power → minimum cost
⚠️
Don't use Opus for everything. A massive refactoring session with Opus can cost 10–20× more than with Sonnet for a similar result on standard tasks. Reserve Opus for when maximum intelligence truly matters.

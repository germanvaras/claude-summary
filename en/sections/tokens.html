🪙 Tokens & Context Fundamental↗ Docs

💡
Key concept: Understanding tokens and context means understanding Claude's real limits. Everything else — skills, CLAUDE.md, subagents — exists partly to manage this finite resource.

🔢 What is a token?

A token is the minimum unit the model processes. It is not a word or a character — it is a fragment that can be part of a word, a whole word, or a punctuation mark. The tokenizer splits text before sending it to the model.

TextApprox. tokensNote
A simple line of code~10–20Varies significantly by language

📌
Claude's tokenizer is optimized for English. The same text in Spanish or other languages generally consumes more tokens than in English — meaning it is less efficient.

📦 The context window

The context window is the model's "workspace" — everything Claude can "see" at once: conversation history, read files, CLAUDE.md, active skills, and tool results. It is finite. Everything shares the same space.

📌
Current Claude models have different context windows: Opus 4.6 and Sonnet 4.6 have 1,000,000 tokens (1M); Haiku 4.5 has 200,000 tokens. This budget covers history, files, CLAUDE.md, and skills (input) plus the generated response (output). In interactive use it is rarely a problem. In pipelines with heavy generation, the per-response output limit is the real practical ceiling: Opus 4.6 up to 128K tokens, Sonnet 4.6 and Haiku 4.5 up to 64K tokens.

🫧 What happens when the context fills up?

Claude does not give an explicit error — what happens is more subtle and harder to detect:

1

Gradual degradation

Claude begins to lose focus on instructions from early in the conversation. CLAUDE.md may fall out of the "effective focus" even though it is technically still loaded. Responses become more generic and less precise.

2

Loss of coherence

Claude begins to contradict itself: it applies different conventions than those established, repeats questions you already answered, or ignores CLAUDE.md restrictions defined at the start of the session.

3

Hard cutoff

In extreme cases, the API rejects the request because it exceeds the maximum limit. An explicit error appears. This is the least frequent case.

⚠️
Claude "forgetting" things in long sessions is almost never a model bug — it is the loaded context. Before assuming Claude "doesn't understand," use /compact to compress the history, or start a new session with just claude. Use claude -c only if you want to resume the previous session where it left off — it is not a new session.

🗜️ /compact — the key tool

The /compact command compresses the conversation history: the model generates a dense summary of what was discussed and replaces the full history with that summary. The result: frees context space without losing the thread of the task.

/compactNew session
Frees context✅ Yes✅ Yes
Keeps task thread✅ Yes (summarized)❌ No
Keeps CLAUDE.md and skills✅ Yes✅ Yes
When to use itLong session with accumulated contextCompletely new task
 
/compact

# You can also pass focus instructions:
/compact focus on the changes in the Orders module
💡
Don't wait for the context to explode. Compact proactively when you feel the session has been running a while and Claude is starting to lose focus — not as a last-resort rescue.

🤖 SubagentsDelegate independent tasks to separate contextsThe main agent doesn't accumulate all the work

✅
Golden rule: The context is a shared resource between you, Claude, and the project. Managing CLAUDE.md, skills, and session length well is just as important as writing good code.

🔌 MCP Servers External tools↗ Docs

💡
Key concept: MCP (Model Context Protocol) is the system that allows Claude to interact with real external tools: databases, APIs, cloud services. Without MCP, Claude only works with local files. With MCP, it can run SQL queries, create GitHub issues, search the web, and much more.

🧩 What MCP really is

MCP is an open standard protocol (published by Anthropic, with a public spec) that defines how language models communicate with external tools. The key idea is standardization: an MCP server works with any compatible client (Claude Code, Claude Desktop, others).

An MCP server is a separate process that exposes capabilities to the model. Claude Code launches it, connects to it, and can invoke its functions during a task. From Claude's perspective, MCP server tools appear as additional tools alongside Read, Write, Bash, etc.

Claude Code (MCP client)
│
├── Native tools: Read, Write, Edit, Bash, Grep...
│
└── MCP tools (via external server):
    ├── query_database(sql)         ← postgres MCP server
    ├── create_issue(title, body)   ← github MCP server
    └── search_web(query)           ← brave-search MCP server

🔌 Transport types

MCP servers can communicate with Claude Code in two ways:

TransportHow it worksWhen to use itConfig
stdio Claude Code launches the server as a child process. They communicate via stdin/stdout (JSON-RPC). The server lives as long as Claude Code runs. Most cases: local servers, npx, binaries command + args
HTTP (Streamable HTTP) The server runs independently (can be remote). Claude Code connects via HTTP POST/GET. Optionally uses SSE for streaming responses. Remote servers, cloud services, shared team APIs. Recommended for remote cases. type: "http" + url + headers
SSE ⚠️ Deprecated Previous transport based on Server-Sent Events. Still works but is deprecated. Only for compatibility with legacy servers that haven't migrated to HTTP yet. type: "sse" + url
📌
For local servers (npx, binaries), use stdio. For remote services or team APIs, use HTTP — it is the recommended transport for cloud use. Avoid SSE in new servers: it is deprecated.

📦 Installing an MCP server

There are three ways to install a server:

A

Via npx (automatic, no prior installation)

The simplest method: set npx as the command and Claude Code downloads the server the first time it is needed.

"command": "npx",
"args": ["-y", "@modelcontextprotocol/server-github"]

The -y automatically accepts the installation. Requires internet access and Node.js.

⚠️
On corporate networks with a proxy, npx may fail. In that case, install the package globally first: npm install -g @modelcontextprotocol/server-github and then use the binary directly.
B

Global npm installation

npm install -g @modelcontextprotocol/server-postgres

More reliable for continuous use. After installing, you can use the command directly in config without npx:

"command": "mcp-server-postgres",
"args": ["postgresql://user:pass@localhost/mydb"]
C

Remote HTTP server

If the server is already running somewhere (your infra, a team service):

"url": "http://mcp-server.internal:8080"

⚙️ Configuration in settings.json

MCP servers are configured in the mcpServers block of settings.json. You can have multiple servers active simultaneously.

Structure for stdio server:

{
  "mcpServers": {
    "server-name": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-NAME"],
      "env": {
        "VARIABLE": "value"
      }
    }
  }
}

Structure for HTTP server:

{
  "mcpServers": {
    "my-remote-server": {
      "type": "http",
      "url": "https://api.example.com/mcp",
      "headers": {
        "Authorization": "Bearer ${API_KEY}"
      }
    }
  }
}
⚠️
Don't put real credentials in the project's .claude/settings.json (it goes to the repository). Use settings.local.json for sensitive values, or system environment variables instead of the env block.

🧱 The three MCP components

An MCP server can expose up to three types of capabilities:

ComponentWhat it isHow Claude uses itExample
Tools Functions Claude can invoke to do something: run queries, create resources, call APIs. Automatically when relevant, or when you explicitly ask. Appear in context just like Read or Bash. execute_query(sql), create_issue(title, body), post_message(channel, text)
Resources Data Claude can read: database records, remote files, schemas. Read-only by definition. Claude requests them to get context. The server returns content injected into Claude's context. Full DB schema, API documentation, last hour's logs
Prompts Predefined prompt templates exposed by the server that appear as slash commands with the prefix /mcp__server__name. The user invokes them manually, just like a custom slash command. /mcp__github__create-pr, /mcp__postgres__explain-query

🔌 Verified available servers

The official npm namespace is @modelcontextprotocol/. Before using any third-party server, verify it exists on npmjs.com.

PackageWhat it providesRequired credential
@modelcontextprotocol/server-postgresPostgreSQL queries and schemaConnection string in args
@modelcontextprotocol/server-githubRepos, issues, PRs, code searchGITHUB_PERSONAL_ACCESS_TOKEN
@modelcontextprotocol/server-slackRead channels, send messagesSLACK_BOT_TOKEN, SLACK_TEAM_ID
@modelcontextprotocol/server-brave-searchWeb search via Brave Search APIBRAVE_API_KEY
@modelcontextprotocol/server-sqliteSQLite queries on local filesPath to .db file in args
@modelcontextprotocol/server-filesystemExtended file system accessAllowed paths in args
📌
There is no official server for SQL Server (MSSQL) under the @modelcontextprotocol/ namespace. For SQL Server, search npmjs.com or consider exposing data via a custom server built with the MCP SDK.

💡 Full working example — GitHub

Configuration in .claude/settings.local.json (does not go to the repo — contains the token):

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_your_token_here"
      }
    }
  }
}

Once connected, Claude can do this automatically:

> "find open issues with the 'bug' label in the current repo"
# Claude invokes: list_issues(labels=["bug"], state="open")
# Shows: 3 issues found with title, number, and description

> "create an issue: 'Fix null reference in OrderService' with label bug"
# Claude invokes: create_issue(title=..., body=..., labels=["bug"])
# Result: Issue #47 created on GitHub

> "show the last merged PR"
# Claude invokes: list_pull_requests(state="closed", sort="updated")
# Shows: PR #46 - "Add CQRS to Orders module"

💡 Full working example — PostgreSQL

Configuration in .claude/settings.local.json:

{
  "mcpServers": {
    "staging-db": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-postgres",
        "postgresql://readonly_user:pass@staging-db:5432/ecommerce"
      ]
    }
  }
}

In the session:

> "how many orders have the 'pending' status in the last 7 days?"
# Claude invokes: query("SELECT COUNT(*) FROM orders WHERE status='pending' AND created_at > NOW()-INTERVAL '7 days'")
# Result: 142 orders

> "show the schema of the orders table"
# Claude invokes: describe_table("orders")
# Shows: columns, types, constraints, indexes

🔧 Debugging the MCP connection

1

Check status from within the session

/mcp

Shows all configured servers with their status: connected, disconnected, or error.

2

View initialization errors

claude --verbose

In verbose mode you'll see exactly why the connection fails: package not installed, wrong credential, unreachable host.

3

Common issues checklist

SymptomLikely causeSolution
Server appears disconnectednpx package not installed / network blockedInstall globally: npm install -g @modelcontextprotocol/server-NAME
Authentication errorMissing or incorrect environment variableVerify the variable is set: echo %GITHUB_PERSONAL_ACCESS_TOKEN%
Server connected but no tools availableError initializing the serverRun claude --verbose and read the server's stack trace
Connection refused (HTTP server)Remote server is not runningCheck with curl http://host:port/health

🏗️ Build your own MCP server

If you need to integrate an API or system that has no available MCP server, you can create your own using the official SDK:

SDKPackage
TypeScript / JavaScript@modelcontextprotocol/sdk
Pythonmcp (PyPI)

The full spec is at modelcontextprotocol.io. The SDK handles the protocol; you only define what tools, resources, and prompts to expose.

📌
A custom MCP server is the right solution when you need to connect Claude to internal systems: your ERP, private APIs, legacy databases, or any service that has no public MCP server.

The most frequent issues when working with Claude Code, organized by symptom. Click each one to see the cause and solution.

🧩 Skills

My skill doesn't activate when I ask for it Skill ▶
🔴
Symptom: Claude does not activate the skill even though the request seems relevant.

Causes and solutions:

  • 🔄 You did not restart Claude Code after creating or editing the skill. Skills load at session start — always close with /exit and reopen with claude after changes.
  • 📝 The description is too vague. If it only says "helps with code," Claude doesn't know when to use it. Add verbs and specific phrases: "Use when reviewing PRs, checking code before merge, or when asked for feedback on a branch."
  • 📁 The file is in the wrong location. The structure must be .claude/skills/skill-name/SKILL.md — not .claude/skills/SKILL.md directly. Each skill needs its own subfolder.
  • ✏️ The frontmatter has errors. The name and description fields are required and must be between the --- dashes exactly.

Quick check:

/skills

If your skill does not appear in the list, the problem is location or frontmatter. If it appears but does not activate, the problem is the description.

The wrong skill activates when I have two with similar names Skill ▶
🔴
Symptom: Claude activates the project skill instead of the personal one, or vice versa.

The priority order is: Managed → Personal → Project → Plugins. If two skills have the same name, the one with higher priority wins.

  • Use descriptive names to avoid collisions: frontend-review instead of review.
  • If you want the project skill to override the personal one, the only option is to rename the personal skill or give it a different name.
  • Use /skills to see the origin (personal/project) of each active skill.
The skill loads but Claude doesn't follow the instructions correctly Skill ▶
🔴
Symptom: Claude confirms it activated the skill but the result does not respect the defined format or steps.
  • 📏 SKILL.md is too long. If it exceeds 500 lines, Claude may skip parts. Move extensive material to files in references/ and reference them from SKILL.md.
  • ⚔️ Conflict with CLAUDE.md. If CLAUDE.md has instructions that contradict the skill, they can compete. Check for overlap.
  • 🔢 Ambiguous instructions. Numbered steps and bullet lists work better than long paragraphs for procedural instructions.

🪝 Hooks

The hook doesn't run after Claude edits a file Hook ▶
🔴
Symptom: Claude writes or edits files but the hook does not fire.
  • 🔄 Restart Claude Code. Hooks are read at session start.
  • 📄 Check the settings.json JSON syntax. A syntax error silences all hooks. Validate with:
claude config list
  • 🎯 Review the matcher. "matcher": "Write" only activates on writes. For edits too, use "matcher": "Write|Edit".
  • 📁 settings.json location. Must be at .claude/settings.json (project) or ~/.claude/settings.json (global).
The hook fails with "command not found" or path errors Hook ▶
🔴
Symptom: The hook fires but the command is not found or fails with a path error.
  • 🛣️ Hooks run with the system PATH, not the interactive shell's PATH. Use absolute paths. On Windows, check with where npm for the full path.
  • 📦 The command must be in the project directory. Prefix with: cd C:\path\project && npm run lint
  • 🔑 Execution permissions. On Linux/macOS, check: chmod +x script.sh
  • 🔍 Use verbose mode to see the exact error:
claude --verbose

🔌 MCP

The MCP server appears as disconnected MCP ▶
🔴
Symptom: /mcp shows the server as offline or Claude says it cannot connect.
  • 📦 The package is not installed. With npx -y it installs automatically, but may fail on corporate networks. Install manually first: npm install -g @modelcontextprotocol/server-NAME
  • 🔑 Missing environment variables. Verify the variables in the env block of settings.json are defined and correct.
  • 🔒 Connectivity issue. If the DB or API is remote, verify the host is accessible from your machine.
  • 📋 Check server logs with verbose:
claude --verbose

In verbose mode you'll see the MCP server initialization error in detail.

📄 CLAUDE.md / General

Claude ignores CLAUDE.md instructions CLAUDE.md ▶
🔴
Symptom: Claude does not respect conventions or restrictions defined in CLAUDE.md.
  • 📏 CLAUDE.md is too long. If it has hundreds of lines, instructions at the end may fall outside the effective focus. Keep it under 200 lines and move extensive material to skills.
  • ⚔️ Contradictory instruction in the chat. If you give a contrary instruction in the same message, it may override CLAUDE.md. CLAUDE.md sets defaults, not absolute rules. For absolute restrictions, use deny in permissions.
  • 📁 File misspelled or mislocated. Must be exactly CLAUDE.md (uppercase) at the project root, or at .claude/CLAUDE.md.
  • 💡 The context filled up. In long sessions, CLAUDE.md may lose effectiveness. Use /compact to compress the history.
The context fills up quickly and Claude starts "forgetting" things General ▶
🔴
Symptom: In long sessions, Claude starts ignoring instructions or losing the thread of the task.
  • 🗜️ Use /compact to compress the conversation history without losing essential context. You can add focus instructions: /compact focus on the changes in the Orders module
  • 📂 Move CLAUDE.md content to skills. What does not need to be always present consumes context unnecessarily.
  • ✂️ Break up long tasks. Instead of one giant session, use subagents or separate sessions for different modules.
  • 🔄 Start a new session with just claude for a clean context. CLAUDE.md and skills reload automatically. Use claude -c to resume the previous session (does not compress — continues where it was).

🔑 Authentication

Claude Code asks to login every time I open it Auth ▶
🔴
Symptom: The session token does not persist between sessions.
  • 📁 The credentials file may have been deleted. Verify it exists: C:\Users\<user>\.claude\credentials.json
  • 🔒 File permissions. The Claude Code process must be able to read and write that file.
  • 🔑 Permanent alternative: Set ANTHROPIC_API_KEY as a system environment variable with setx. No browser login needed.

💡
Key concept: An agent is not just Claude answering questions — it is Claude executing tasks autonomously, making decisions, and using tools to reach a concrete result without continuous user intervention.

📌 What is an Agent?

In Claude Code, an agent is a Claude session that can invoke tools, read and write files, execute commands, and chain multiple actions to complete a complex task. The difference from a regular chat is autonomy: the agent decides what to do at each step, it does not wait for instruction by instruction.

🔄 Operating cycle

  • 🎯 Receives a task — from the user or delegated by another agent.
  • 🗺️ Plans — breaks the task down into executable steps.
  • ⚙️ Executes tools — reads files, runs commands, calls APIs.
  • 🔍 Evaluates results — analyzes the output of each action.
  • 🔁 Iterates — adjusts the plan based on what it finds, until done.

⚖️ Skill vs. Agent — the key difference

📌
The most common confusion: a skill teaches Claude how to do something. An agent does it. The skill provides the knowledge; the agent executes with autonomy. They are complementary, not alternatives.
🧩 Skill🤖 Agent
What is it?Instructions in SKILL.md that Claude reads before actingClaude in action: plans, decides, executes, iterates
How does it activate?Automatically by semantic contextAny Claude Code session is already an agent
Does it modify files?Not directly — it provides the procedureYes, uses Read / Write / Edit / Bash autonomously
Does it make decisions?No — it is passive, only guidesYes — chooses which tool to use and when
Own context?No, lives in the context of the agent that loaded itYes, each agent has its own context
Use case"Follow this checklist when reviewing PRs""Refactor the entire Orders module"
✅
The ideal combination: Skill + Agent. The skill gives the specific context ("how to apply CQRS in this project") and the agent executes it autonomously ("refactor OrderService.cs"). Without a skill, the agent improvises. Without an agent, the skill does nothing on its own.

🌳 Agent tree — orchestration

When a task is too large for a single context, or has independent parts that can run in parallel, the main agent can delegate work to subagents. This forms an execution tree:

Main Agent  →  "Migrate the Orders module to CQRS"
│
├── Subagent A  →  "Refactor OrderService.cs"
│                    isolated context · works alone · returns result
│
├── Subagent B  →  "Write tests for CreateOrderCommandHandler"
│                    isolated context · works alone · returns result
│
└── Subagent C  →  "Review the affected EF Core migrations"
                     isolated context · works alone · returns result

The main agent does not execute the changes directly — it delegates, receives the results from each subagent, and integrates or validates them. Each subagent works without knowing the others exist.

⚠️
Subagents do not share context. They do not see the main agent's conversation or each other's. They only receive what the main agent explicitly passes in the delegation instruction.

🤖 Subagents — isolated context

A subagent is an agent launched by the main one with a specific delegated task. Its main advantage is isolation: errors, history, or state of a subagent do not affect the main agent or other subagents.

CharacteristicDetail
Own contextDoes not see the main agent's history or other subagents
Receives only what's neededThe main agent passes exactly the instructions and relevant files for its task
Runs in parallelMultiple subagents can run simultaneously on different parts of the problem
Returns resultsIts output (modified files, reports, etc.) returns to the main agent when done
No contaminationAn error or loaded context in a subagent does not affect the main agent's context
💡
When to use subagents: When the task has independent parts, when the main context is filling up with work that could be isolated, or when you want different parts of the task to use different skills without interference.

🛠️ Available tools

ToolWhat it does
ReadReads the content of a file
WriteWrites or overwrites a file
EditModifies specific parts of a file
BashExecutes shell commands
GrepSearches for patterns in files
GlobLists files by pattern
AgentLaunches a subagent with isolated context and delegated task
⚠️
An agent with unrestricted access to Write and Bash can modify any file in the project. Always define the minimum necessary scope using allowed-tools in skills or permissions in settings.json.

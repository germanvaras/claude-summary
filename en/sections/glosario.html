Terms from the Claude Code ecosystem — everything you need to read this documentation without searching every other line.
Context Window
The maximum number of tokens Claude can process in an interaction. This budget covers input + output combined: history, CLAUDE.md, skills, read files (input) and the generated response (output). Models Opus 4.6, Sonnet 4.6, and Haiku 4.5 have 200,000 tokens. The maximum output per response is a separate, substantially lower limit.
If the context fills up, Claude loses focus on instructions from early in the conversation — not a bug, it is the context window limit
Token
The basic unit language models use to process text. Not a word or a character — it is a fragment that can be part of a word, a whole word, or a punctuation mark. Approximately 3–4 characters in English. In Spanish the tokenizer is less efficient — the same text consumes more tokens than in English.
A 500-word CLAUDE.md consumes ~700 tokens of context in every conversation
Frontmatter
Metadata block at the start of a Markdown file, delimited by ---. In Claude Code it is used to define the name, description, and optional fields of skills and custom slash commands.
The --- name: pr-review --- block at the start of a SKILL.md is the frontmatter
Semantic Matching
The mechanism by which Claude compares your request with the description field of available skills to decide which one to activate. It does not look for exact words — it interprets the intent and meaning of what you ask. That is why the skill's description is so important.
"review the PR" can activate a skill described as "reviews code changes before merge"
Skill
A SKILL.md file inside its own folder that teaches Claude how to perform a specific task. It activates automatically when Claude detects that your request semantically matches its description. No manual invocation required.
A pr-review skill activates on its own when you ask to review a PR
Agent
A Claude instance that operates with autonomy within an execution context: it can invoke tools, read and write files, execute commands, and chain actions to complete a complex task without continuous intervention.
Claude refactoring an entire module on its own, file by file, verifying it compiles
Subagent
A Claude instance launched by the main agent to execute a delegated task in a completely isolated context. It receives instructions, works independently, and returns results without "seeing" the main conversation.
The main agent delegates "write the tests" to a subagent while continuing another task
Execution Context
The isolated environment in which an agent or subagent operates: which tools are available, what instructions were received, and the state of its conversation. Each subagent has its own, separate from the main agent's.
The tests subagent does not "see" the main agent's conversation history
MCP (Model Context Protocol)
An open standard protocol that defines how language models communicate with external tools and services. Allows Claude to connect to databases, APIs, and services without custom integration code. The official server namespace on npm is @modelcontextprotocol/.
A PostgreSQL MCP server allows Claude to execute SQL queries directly against a database
MCP Server
An external process that implements the MCP protocol and exposes capabilities (Tools, Resources, Prompts) that Claude can use during a task. Configured in settings.json and runs as a separate process when Claude Code starts. Can use stdio transport (local process) or HTTP (remote server).
@modelcontextprotocol/server-github exposes the GitHub API as tools Claude can invoke
Hook
A script or command that Claude Code executes automatically when a specific event occurs — without the user asking. Hooks receive event information via stdin (JSON) and can allow or block actions via their exit code.
A PostToolUse hook runs the linter every time Claude saves a file
Matcher
A regular expression (regex) in the hook configuration that defines which tool or event it matches against. The | character is the regex OR operator. Omitting the field makes the hook fire on any tool.
"matcher": "Write|Edit" activates the hook when Claude writes or edits files, but not on Bash or Read
Slash Command
A predefined instruction the user invokes manually by typing /name in the conversation. Three types exist: built-in (system), skill commands (created by you in .claude/commands/), and MCP prompts (exposed by MCP servers).
/coverage-report generates the coverage report when the developer decides to run it
$ARGUMENTS
A special variable available in custom slash command files. Captures all the text the user writes after the command name. Enables creating parameterized commands.
In /review feature/login, $ARGUMENTS captures the value feature/login
Progressive Disclosure
An organization technique where only essential content is loaded initially, and additional detail is incorporated on demand. In skills: SKILL.md has the essentials, the references/ folder has the extensive material that only loads when needed.
The 200-line architecture guide only loads into context when someone asks about architecture
allowed-tools
An optional field in a skill's frontmatter that restricts which native tools Claude can use while that skill is active. Useful for creating read-only skills or skills with security guardrails.
allowed-tools: Read, Grep prevents Claude from writing files while the skill is active
Settings Cascade
Claude Code's configuration resolution system, where multiple settings.json files are combined by level. The precedence order (highest to lowest) is: Managed → Local → Project → User. Rules from the most specific level take precedence.
A rule in .claude/settings.local.json (Local) overrides the configuration in .claude/settings.json (Project)
Worktree
An additional working directory linked to the same Git repository, allowing multiple branches to be active simultaneously. Claude Code uses the --worktree flag to create an isolated worktree where the agent works without affecting your main directory.
claude --worktree creates an isolated repo copy for Claude to do refactoring without touching your current work
Headless Mode / Print Mode
Claude Code's non-interactive execution mode, activated with claude -p "task". Sends the prompt, waits for the full response, and exits. Does not open an interactive session. Designed for use in scripts, CI/CD pipelines, and automations.
claude -p "generate the changelog for version 2.4.0" --output-format json returns the result as JSON and exits
/doctor
Claude Code's diagnostic slash command that checks the environment state: installed version, authentication, API connectivity, and active configuration. First step for any technical issue.
If Claude Code fails silently, open a session and run /doctor to see the diagnostic

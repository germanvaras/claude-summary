🗂️ Configuration Scopes Reference↗ Docs

💡
Key concept: Claude Code has four configuration levels. Understanding which applies in each context defines what gets committed to the repo, what is personal, and what the organization manages.

📐 The four scopes

ScopeFileCommitted?Who controls it
🏢 Managed Central organization configuration — IT / Admins. Overrides everything. Not editable by the user.
👤 User ~/.claude/settings.json ❌ No — it's personal You. Applies to all your projects. Personal preferences, API keys, global hooks.
📁 Project .claude/settings.json ✅ Yes — goes to the repo The team. Hooks, MCP, permissions, and shared skills. Inherited by everyone who clones.
🔒 Local .claude/settings.local.json ❌ No — Claude Code adds it to .gitignore automatically You, per project. Higher priority than Project and User. Local overrides: connection strings, dev tokens, personal per-repo prefs.
⚠️
Never put secrets (tokens, passwords, real connection strings) in .claude/settings.json — that file goes to the repository. Always use .claude/settings.local.json for sensitive values.

🔁 Precedence order

When there is a conflict between scopes, the precedence order (highest to lowest) is:

Managed  >  Local  >  Project  >  User

Local overrides Project and User — it is the personal per-repository override. A deny rule in Managed cannot be overridden by any other scope.

📋 What goes in each scope — practical guide
